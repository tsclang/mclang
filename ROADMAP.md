# MClang — Дорожная карта реализации

**Язык реализации:** TypeScript (Node.js)

Каждая фаза заканчивается тестируемым результатом. Следующая фаза начинается только после прохождения теста предыдущей.

## Фаза 0 — Инициализация проекта

**Цель:** настроить окружение TypeScript.

### Задачи

- `npm init` + `tsconfig.json`
- Зависимости: `typescript`, `ts-node`, `@types/node`
- Тест-фреймворк: `vitest`
- Структура папок: `src/lexer`, `src/parser`, `src/ast`, `src/codegen`, `src/math`, `src/types`, `src/cli`
- CLI-точка входа: `src/cli/index.ts`
- Скрипты в `package.json`: `build`, `test`, `dev`

### Критерий готовности

`npm run build` проходит без ошибок. `npx ts-node src/cli/index.ts --help` выводит справку.

## Фаза 1 — Лексер

**Цель:** превратить текст `.mc`-файла в поток токенов.

### Задачи

- Токенизация чисел (целые, дробные, научная нотация `1e-9`)
- Токенизация ASCII-идентификаторов
- Токенизация Unicode-идентификаторов (греческие буквы, `π`, `α`, …)
- Трансляция LaTeX-команд в токены:
  - `\lfloor` → `⌊`, `\rfloor` → `⌋`
  - `\lceil` → `⌈`, `\rceil` → `⌉`
  - `\lVert` → `‖`, `\rVert` → `‖`
  - `\pm`, `\cdot`, `\times`, `\in`, `\notin`, `\to`, `\infty` и др.
- Таблица синонимов trig-функций (`tg` → `tan`, `sh` → `sinh`, …)
- Disambiguation `|` vs `||` по контексту предыдущего токена
- Lookahead: `\sigma{` → функция, `\sigma` → идентификатор
- Распознавание `INDENT` / `DEDENT`
- Символ `°` → токен `DEGREE`
- Постфикс `!` → токен `FACTORIAL`
- Пропуск комментариев `//`

### Критерий готовности

Лексер на входе получает `.mc`-файл и выдаёт корректный поток токенов с типами и позициями. Покрытие: все токены из PEG-грамматики.

## Фаза 2 — Парсер и AST

**Цель:** построить AST из потока токенов.

### Задачи

- Рекурсивный нисходящий PEG-парсер
- Узлы AST: `Number`, `Identifier`, `BinaryOp`, `UnaryOp`, `FuncCall`, `FuncDef`, `ConstDef`, `Block`, `IfStmt`, `ForStmt`, `WhileStmt`, `WhereBlock`, `IndexExpr`, `SliceExpr`, `ArrayLit`, `MatrixLit`
- Объявление функций с параметрами и типами
- Глобальные константы
- Параметры по умолчанию
- `where`-блок: определения (топосорт) и guard-и (по порядку)
- Неявный `return` (последнее выражение)
- ASI (автоточки с запятой)
- Значимые отступы

### Критерий готовности

Парсер строит AST для полного примера из раздела 22 SPEC.md без ошибок. AST можно распечатать в читаемом виде.

## Фаза 3 — Базовый генератор Си

**Цель:** из AST получить валидный `.c` + `.h` файл для простых функций.

### Задачи

- Генерация `mc_num f(mc_num x, mc_num y) { return ...; }`
- Генерация заголовочного файла `.h` (только публичные функции, без `_`-префикса)
- Макрос `mc_num` в `.h`-файле
- Трансляция бинарных и унарных операций
- Трансляция вызовов функций
- Трансляция `if/else` (инлайн + блочный + guard clause)
- Трансляция `\begin{cases}`
- Трансляция `for..in` и `while`
- Трансляция `where`-блоков

### Критерий готовности

```
mclang physics.mc --target c
```
Компилируется `gcc`/`clang` без предупреждений. Результат вызова функций совпадает с эталонными значениями.

## Фаза 4 — Математический движок

**Цель:** поддержка LaTeX-синтаксиса и математических конструкций.

### Задачи

- Неявное умножение: `2x`, `2(a+b)`, `2πr`
- `\frac{a}{b}` → `((a) / (b))`
- `\sqrt{x}`, `\sqrt[n]{x}`
- Степень `x^n`, `x**n` → `pow(x, n)`
- Все trig-функции + русские синонимы
- Логарифмы: `ln`, `log`, `log10`, `\lg`, `\log_{b}{x}`
- `exp(x)`, `e^x`
- `|x|`, `\abs{x}`, `\lfloor x \rfloor`, `\lceil x \rceil`, `\lVert v \rVert`
- Встроенные константы: `π`, `e`, `τ`, `φ`, `inf`, `nan`
- Constant folding для константных выражений
- Транслитерация Unicode-идентификаторов → ASCII (`π` → `__uni_pi`)
- `\sigma{v}` → `std(v)`, `\Gamma{x}` → `tgamma(x)`, `\bar{v}` → `mean(v)`
- Цепочки сравнений: `0 < x < 10`
- Символ `°` → умножение на `π/180`
- `\neq`, `\leq`, `\geq` как синонимы операторов

### Критерий готовности

Гауссова функция, формула Ньютона, квадратное уравнение компилируются и дают верный результат.

---

## Фаза 5 — Типы, массивы и матрицы

**Цель:** поддержка векторных и матричных вычислений.

### Задачи

- Типы `num[]`, `num[][]`, `num[N]`
- Неявные параметры `_len`, `_rows`, `_cols`
- `.length`, `.rows`, `.cols`
- Runtime-guard при несовпадении длин массивов
- Литералы `[1, 2, 3]`, `[[1,0],[0,1]]`
- Индексация `v[i]`, `m[i][j]`
- Срезы `v[a..b]`, `m[:, j]`
- Плоское хранение матриц: `m[i][j]` → `m[i * m_cols + j]`
- Dispatch `⋅` по типу: скаляр / dot product / матричное умножение
- Поэлементные операции: `+`, `-`, `.*`
- Векторное произведение `⨯`
- Функции: `transpose`, `det`, `inv`, `norm`
- Константы: `I(n)`, `zeros(r,c)`, `ones(r,c)`
- Множественный возврат `\pm expr` → `num[2]`

### Критерий готовности

Умножение матриц, нормализация вектора, отражение вектора компилируются и дают верный результат.

---

## Фаза 6 — Агрегаторы и специальные операторы

**Цель:** суммы, произведения, статистика, числовая теория.

### Задачи

- `\sum_{i=a}^{b}`, `\prod_{i=a}^{b}` — по диапазону
- `\sum_{x \in v}`, `\prod_{x \in v}`, `\min_{x \in v}`, `\max_{x \in v}` — по массиву
- Агрегаторы: `sum(v)`, `product(v)`, `mean(v)`, `std(v)`, `min(v)`, `max(v)`
- Dispatch `min`/`max` по числу аргументов
- `n!` → цикл или lookup-таблица (до `\leq 20`)
- `\binom{n}{k}`
- `\gcd(a,b)` → алгоритм Евклида
- `\lcm(a,b)`
- `sgn(x)`
- `erf(x)`, `gamma(x)`

### Критерий готовности

Факториал, биномиальный коэффициент, статистика массива компилируются и верны.

## Фаза 7 — Численные методы

**Цель:** пределы, производные, интегралы, решение уравнений.

### Задачи

- `\lim{x \to a} expr` → вычисление в `a ± 1e-9`
- `\lim{x \to \infty} expr` → вычисление при `x = 1e15`
- `\frac{d}{dx} expr` → конечные разности `(f(x+h) - f(x)) / h`
- `\int{a}{b} expr dx` → метод Симпсона через `mc_integrate`
- `solve(t, lo, hi) { expr == val }` → бисекция, точность `1e-9`, нет решения → `nan`
- `is_nan(x)`, `is_inf(x)`, `is_finite(x)`

### Критерий готовности

Интеграл `sin(x)` от 0 до π ≈ 2.0, производная `x²` в точке 3 ≈ 6.0, `solve` находит корень `x² - 2 = 0`.

## Фаза 8 — Таблицы и данные

**Цель:** поддержка lookup-таблиц и интерполяции.

### Задачи

- Объявление числовых таблиц (массивы пар)
- Вызов таблицы как функции → линейная интерполяция
- Именованные таблицы со строковыми ключами (compile-time)
- `import_table("file.csv")` → статический массив в Си

### Критерий готовности

Таблица `steam_data` интерполирует значение между двумя точками.

---

## Фаза 9 — Таргеты и сборка

**Цель:** финальная генерация под все платформы.

### Задачи

- `--target c` — финальная полировка `.c` + `.h`
- `--target wasm` — интеграция с Emscripten
- `--target shared` — Python C extension (`.so` / `.dll`)
- `--precision f64` (default), `--precision f32`, `--precision fixed`
- Стандартная библиотека: физические константы (`G`, `c`, `h`, `k_B`, …) как встроенные

### Критерий готовности

`physics.mc` компилируется и вызывается из JS (Wasm), Python и Си без модификаций.
