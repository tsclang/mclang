# Constant folding

Constant folding — оптимизация компилятора: вычисление выражений, состоящих только из констант, на этапе компиляции, а не во время выполнения.

---

## Что сворачивается

mclang сворачивает:
- Арифметические выражения из литералов: `2 + 3` → `5`
- Выражения с встроенными константами: `2 * π` → `6.283185…`
- Степени: `2^10` → `1024`
- Деление: `355 / 113` → `3.141592…`
- Унарный минус: `-1` → `-1`

---

## Примеры

```mc
// Значения вычисляются при компиляции
A = 2 * π          // → 6.283185307179586
B = 180 / π        // → 57.29577951308232
C = sqrt(2)        // → 1.4142135623730951 (если аргумент константа)
D = 1 + 2 + 3 + 4  // → 10
```

Генерируемый Си-код:

```c
static const mc_num A = 6.283185307179586;
static const mc_num B = 57.29577951308232;
static const mc_num D = 10.0;
```

---

## Folding в телах функций

```mc
to_radians(deg) = deg * (π / 180)
```

`(π / 180)` — константа, вычисляется при компиляции:

```c
mc_num to_radians(mc_num deg) {
    return deg * 0.017453292519943295;
}
```

---

## Что НЕ сворачивается (MVP)

- Вызовы тригонометрических функций с константами (кроме `sqrt`)
- Выражения, зависящие от параметров
- Условные выражения (`if`)

```mc
A = sin(π / 6)   // NOT folded в MVP → остаётся sin(M_PI / 6.0)
```

Полный CAS с аналитическими вычислениями — в v2.

---

## Folding и деление на ноль

```mc
A = 1 / 0   // ошибка компиляции: деление на ноль при сворачивании
```
