# Приоритет имён констант

При совпадении имён константы разрешаются по приоритету: **локальные > глобальные > встроенные**.

---

## Иерархия приоритетов

```
1. Локальные переменные и параметры функции
2. Глобальные константы (объявленные в файле или импортированные)
3. Встроенные константы (π, e, τ, φ, inf, nan)
```

---

## Пример 1 — параметр перекрывает глобальную константу

```mc
G = 6.674e-11   // глобальная

force(G, m1, m2, r) =
    G * m1 * m2 / r^2   // здесь G — параметр, не глобальная константа
```

Предупреждение: компилятор сообщит, что параметр `G` перекрывает глобальную константу `G`.

---

## Пример 2 — where-переменная перекрывает глобальную

```mc
g = 9.81   // глобальная гравитация

fall(h) =
    t
    where
        g = 1.62        // локальная — перекрывает глобальную g
        t = sqrt(2*h/g) // используется локальная g = 1.62 (лунная гравитация)
```

---

## Пример 3 — глобальная перекрывает встроенную

```mc
e = 2.718   // перекрывает встроенную e

f(x) = e^x  // использует пользовательскую e = 2.718
```

Компилятор выдаёт предупреждение — это допустимо, но нежелательно.

---

## Рекомендации

- Не перекрывай встроенные константы (`π`, `e`, `nan`) — это вызовет путаницу
- Если надо задать другое значение, используй другое имя: `g_moon = 1.62` вместо `g = 1.62`
- При импорте двух файлов с одинаковыми константами — используй `import as ns`

---

## Ошибки

### Неоднозначность при импорте

```mc
import "./a.mc"   // содержит G = 6.67e-11
import "./b.mc"   // содержит G = 9.81

// Ошибка: G объявлена в двух импортированных файлах
```

**Исправление:**

```mc
import "./a.mc" as phys
import "./b.mc" as earth

f(m) = phys.G * m
```
