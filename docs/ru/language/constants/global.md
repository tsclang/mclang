# Глобальные константы

Константы верхнего уровня — именованные значения, доступные во всех функциях файла.

---

## Синтаксис

```mc
ИМЯ = выражение
```

Объявляется на верхнем уровне, вне функций:

```mc
G = 6.674e-11
c = 2.998e8
h = 6.626e-34

gravitational_force(m1, m2, r) = G * m1 * m2 / r^2
photon_energy(f) = h * f
```

---

## Правила

- Значение должно быть вычислимым в compile time (константное выражение)
- Нельзя ссылаться на параметры функций
- Нельзя ссылаться на другие глобальные константы, объявленные ниже (порядок важен)
- Имя не должно совпадать с именем функции в том же файле

---

## Генерируемый Си-код

```mc
G = 6.674e-11
```

Генерируется как `#define` или `static const`:

```c
static const mc_num G = 6.674e-11;
```

---

## Пример: физические константы

```mc
// constants.mc
G  = 6.674e-11    // гравитационная постоянная
c  = 2.998e8      // скорость света
k  = 1.380e-23    // постоянная Больцмана
Na = 6.022e23     // число Авогадро

// physics.mc
import "./constants.mc"

kinetic(m, v) = 0.5 * m * v^2
rest_energy(m) = m * c^2
```

---

## Константы и constant folding

Если функция использует только глобальные константы и литералы, результат может быть свёрнут:

```mc
TWO_PI = 2 * π
HALF_PI = π / 2

sin_quarter = sin(HALF_PI)   // → 1.0 на этапе компиляции (constant folding)
```

Подробнее: [Constant folding](folding.md).

---

## Ошибки

### Рекурсивная константа

```mc
A = B + 1
B = A - 1   // ошибка: циклическая зависимость
```

### Константа ссылается на параметр функции

```mc
// Нельзя: параметр f недоступен на уровне константы
ENERGY = h * f   // ошибка
```

**Исправление:** сделай это функцией:

```mc
photon_energy(f) = h * f
```
