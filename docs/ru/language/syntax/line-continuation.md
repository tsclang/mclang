# Перенос строк

Длинное выражение можно разбить на несколько строк. mclang не требует явного символа переноса — поведение определяется последним токеном строки (см. [ASI](asi.md)).

---

## Способ 1 — оператор в конце строки

Если строка оканчивается бинарным оператором, следующая строка считается продолжением:

```mc
result =
    a * cos(theta) +
    b * sin(theta) +
    c
```

Работают все бинарные операторы: `+`, `-`, `*`, `/`, `^`, `==`, `<`, `and`, `or` и т.д.

---

## Способ 2 — открытая скобка

Если строка оканчивается на `(` или `[`, продолжение идёт до закрывающей скобки:

```mc
y = clamp(
    x,
    lo,
    hi
)
```

```mc
coords = [
    x0, y0,
    x1, y1,
    x2, y2
]
```

---

## Способ 3 — открытая фигурная скобка LaTeX

```mc
f(x) = \frac{
    x^2 + 1
}{
    x - 1
}
```

---

## Нельзя переносить

Разрыв внутри идентификатора, числа или ключевого слова — ошибка лексера:

```mc
// Плохо: sin разбит на две строки (невозможно, это один токен)
```

---

## Перенос в цепочке сравнений

```mc
valid =
    0.0 <=
    x <=
    1.0
```

---

## Пример: длинная формула

```mc
quadratic(a, b, c) =
    (-b + sqrt(b^2 - 4*a*c)) /
    (2 * a)
```

```mc
heat_transfer(k, A, T1, T2, d) =
    k * A *
    (T1 - T2) /
    d
```

---

## Ошибки

### Разрыв перед оператором без открытой скобки

```mc
// Плохо: ASI вставит ; после "a", и "- b" станет отдельным выражением
result = a
    - b
```

**Исправление:** перенеси оператор на конец предыдущей строки:

```mc
result = a -
    b
```
