# Автоматические точки с запятой (ASI)

mclang автоматически вставляет точки с запятой (`;`) при переходе на новую строку в определённых контекстах. Явно писать `;` не нужно.

---

## Когда `;` вставляется

Точка с запятой вставляется после токена, которым **может** заканчиваться выражение или оператор:

| Последний токен строки | `;` вставляется? |
|------------------------|-----------------|
| Число, идентификатор, `true`, `false` | да |
| `)`, `]`, `}` | да |
| Постфиксные операторы `!`, `°` | да |
| Оператор (`+`, `-`, `*`, …) | **нет** |
| Открывающая скобка `(`, `[` | **нет** |
| Ключевое слово `else`, `where` | **нет** |

---

## Примеры

### Несколько операторов

```mc
f(x) =
    a = x + 1     // ← ASI вставляет ;
    b = a * 2     // ← ASI вставляет ;
    b
```

### Многострочное выражение — оператор в конце строки

```mc
// + в конце строки → продолжение
result = a +
         b +
         c
```

### Многострочное выражение — открывающая скобка

```mc
// ( в конце строки → продолжение
result = sin(
    x * pi / 180
)
```

---

## Что не разрывается

```mc
// цепочка сравнений на нескольких строках — правильно
valid =
    0 <
    x <
    10
```

```mc
// вызов функции со многими аргументами
y = clamp(
    x,
    -1.0,
    1.0
)
```

---

## Ошибки

### Случайный разрыв

```mc
// Плохо: - воспринимается как унарный минус на следующей строке
result = a
    - b        // парсер читает это как: result = a; -b (выражение-заглушка)
```

**Исправление:**

```mc
result = a -
    b
```
