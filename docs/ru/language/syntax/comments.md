# Комментарии

mclang поддерживает два синтаксиса однострочных комментариев. Многострочных комментариев нет.

---

## Синтаксис

### `//` — C-стиль

```mc
// Площадь круга
circle_area(r) = π * r^2
```

### `#` — Python-стиль

```mc
# Площадь круга
circle_area(r) = π * r^2
```

Оба варианта эквивалентны. Можно использовать любой.

---

## Встроенный комментарий

Комментарий можно ставить в конце строки:

```mc
G = 6.674e-11  // гравитационная постоянная (м³ кг⁻¹ с⁻²)

force(m1, m2, r) =
    G * m1 * m2 / r^2   # закон всемирного тяготения
```

---

## Комментарий на отдельной строке

```mc
// ──────────────────────────────
// Баллистические функции
// ──────────────────────────────

range(v0, angle) =
    v0^2 * sin(2 * angle) / g
    where
        g = 9.81  # ускорение свободного падения

# time_of_flight — время до приземления
time_of_flight(v0, angle) =
    2 * v0 * sin(angle) / g
    where
        g = 9.81
```

---

## Комментарии не влияют на ASI и отступы

Строка, состоящая только из комментария, игнорируется лексером при расчёте INDENT/DEDENT. Пустые строки и строки-комментарии не разрывают блок.

```mc
f(x) =
    a = x + 1

    // промежуточное вычисление
    b = a * 2   // это всё ещё внутри f

    b
```

---

## Нет многострочных комментариев

`/* ... */` не поддерживается. Для блочного комментария используй несколько строк:

```mc
// Функция вычисляет угол в радианах
// из градусов, минут и секунд.
// Формула: deg + min/60 + sec/3600, затем * π/180
dms_to_rad(deg, min, sec) =
    (deg + min / 60 + sec / 3600) * π / 180
```
