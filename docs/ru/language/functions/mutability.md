# Мутабельность

В mclang параметры функции **иммутабельны** — их нельзя переназначить внутри тела. Локальные переменные мутабельны.

---

## Параметры — иммутабельны

```mc
f(x) =
    x = x + 1   // ошибка компиляции: параметр x иммутабелен
    x
```

**Исправление:** заведи новую переменную:

```mc
f(x) =
    y = x + 1
    y
```

---

## Локальные переменные — мутабельны

Переменные, объявленные внутри тела функции, можно изменять:

```mc
accumulate(n) =
    acc = 0
    for i in 1..n
        acc = acc + i
    acc
```

`acc` — локальная переменная, можно присваивать внутри цикла.

---

## Where-переменные — иммутабельны

Переменные в блоке `where` вычисляются один раз и не могут быть переназначены:

```mc
f(x) =
    result
    where
        result = x^2
        result = result + 1   // ошибка: where-переменные иммутабельны
```

---

## Почему параметры иммутабельны

1. **Ясность кода** — вызывающий код знает, что переданные значения не изменяются
2. **Си-совместимость** — в Си параметры функции — значения, а не ссылки; переназначение создало бы путаницу
3. **Математический стиль** — в математике `f(x) = expr` подразумевает, что `x` — константа в области видимости функции

---

## Генерируемый Си-код

```mc
f(x) =
    y = x + 1
    y * 2
```

```c
mc_num f(mc_num x) {
    mc_num y = x + 1.0;
    return (y * 2.0);
}
```

Параметр `x` в Си-коде не изменяется — используется напрямую.

---

## Ошибки

### Попытка инкремента параметра

```mc
shift(x, delta) =
    x = x + delta    // ошибка: x — параметр
    x
```

```
Error E042: cannot assign to parameter 'x'
```

**Исправление:**

```mc
shift(x, delta) =
    result = x + delta
    result
```

### Использование до инициализации

```mc
f(x) =
    y = y + 1   // ошибка: y используется до первого присваивания
    y
```

**Исправление:**

```mc
f(x) =
    y = 0
    y = y + 1
    y
```
