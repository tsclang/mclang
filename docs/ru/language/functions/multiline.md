# Многострочное тело функции

Когда тело функции состоит из нескольких выражений или операторов, используется блочный синтаксис с отступом.

---

## Синтаксис

```mc
f(x) =
    оператор1
    оператор2
    ...
    результат
```

Последнее выражение в блоке становится возвращаемым значением (неявный `return`).

---

## Присваивание локальных переменных

```mc
normalize(x, lo, hi) =
    range = hi - lo
    (x - lo) / range
```

`range` — локальная переменная, не попадает в `.h`.

---

## Условные операторы

```mc
abs_value(x) =
    if (x >= 0) x
    else -x
```

```mc
clamp(x, lo, hi) =
    if (x < lo) lo
    else if (x > hi) hi
    else x
```

---

## Циклы

```mc
sum_squares(n) =
    acc = 0
    for i in 1..n
        acc = acc + i^2
    acc
```

---

## Смешанный блок: операторы + where

`where` стоит **после** основного тела:

```mc
trajectory_height(v0, angle, t) =
    vy0 * t - 0.5 * g * t^2
    where
        vy0 = v0 * sin(angle)
        g   = 9.81
```

---

## Вложенные if

```mc
sign(x) =
    if (x > 0) 1
    else if (x < 0) -1
    else 0
```

---

## Ранний выход (guard)

```mc
safe_div(a, b) =
    if (b == 0) nan
    a / b
```

Если `b == 0` — функция вернёт `nan`. Остальной код не выполняется.

Другой способ через `where` guard:

```mc
safe_div(a, b) =
    a / b
    where
        b != 0    // guard: при нарушении вернёт NAN
```

---

## Ошибки

### Нет результирующего выражения

```mc
f(x) =
    a = x + 1
    // нет возвращаемого выражения → ошибка компиляции
```

**Исправление:** добавь выражение-результат в конец:

```mc
f(x) =
    a = x + 1
    a
```

### Переназначение параметра

```mc
f(x) =
    x = x + 1   // ошибка: параметры иммутабельны
    x
```

**Исправление:** используй новую переменную:

```mc
f(x) =
    y = x + 1
    y
```
