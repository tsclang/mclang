# Тип int

`int` — целочисленный тип mclang. Используется **только** для индексов, счётчиков и размеров. Не предназначен для вычислений.

---

## Соответствие в Си

```c
int   // стандартный 32-битный int
```

---

## Когда использовать int

- Параметр-индекс для доступа к массиву: `v[i]`
- Параметр-размер: `n` в `zeros(n, n)`
- Переменная-счётчик в цикле `for`
- Результат функции, возвращающей целое число

```mc
nth_fibonacci(n: int) = ...
```

---

## Объявление параметра

```mc
factorial(n: int) = n!
repeat(x, times: int) = ...
```

---

## Переменная типа int

```mc
f(n: int) =
    count: int = 0
    for i in 1..n
        count = count + 1
    count
```

---

## Неявный int в for

Переменная цикла `for i in a..b` всегда имеет тип `int`:

```mc
sum(n) =
    acc = 0
    for i in 1..n   // i: int
        acc = acc + i
    acc
```

Генерируемый Си-код:

```c
mc_num sum(mc_num n) {
    mc_num acc = 0.0;
    for (int i = 1; i <= (int)n; i++) {
        acc = acc + (mc_num)i;
    }
    return acc;
}
```

---

## Ограничения

- Нельзя использовать `int` как основной тип вычислений (нет `sin(n: int)`)
- Деление двух `int` может вести себя как целочисленное в Си — используй `num`
- Нет арифметики с переполнением — значения, выходящие за `int`, не определены

---

## Смешение num и int

При смешении `int` и `num` в выражениях `int` неявно приводится к `num`:

```mc
f(n: int, x: num) = x + n   // n → (mc_num)n
```

---

## Ошибки

### int как индекс num-выражения

```mc
v(i: num) = arr[i]   // ошибка: индекс должен быть int
```

**Исправление:**

```mc
v(i: int) = arr[i]
```
