# Порядок вычислений в where

Компилятор mclang автоматически упорядочивает переменные в блоке `where` по зависимостям (топологическая сортировка). Guards всегда выполняются в порядке их объявления в исходном коде.

---

## Переменные — топологическая сортировка

Порядок строк в `where` для переменных не важен:

```mc
f(r) =
    surface + volume
    where
        surface = 4 * π * r^2
        volume  = (4/3) * π * r3
        r3      = r^3
```

Компилятор вычислит `r3` раньше `volume`, так как `volume` зависит от `r3`.

Итоговый порядок в Си-коде:
1. `r3 = r^3`
2. `surface = 4 * π * r^2`
3. `volume = (4/3) * π * r3`
4. `return surface + volume`

---

## Guards — порядок объявления

Guards выполняются **строго сверху вниз** в том порядке, в котором написаны:

```mc
check(x, y) =
    x / y
    where
        x >= 0        // guard 1 — проверяется первым
        y != 0        // guard 2 — проверяется вторым
        x < 1000      // guard 3 — проверяется третьим
```

Это позволяет писать guards от «грубых» к «тонким».

---

## Смешанный порядок

При смешивании переменных и guards:

- Guards сохраняют порядок объявления
- Переменные топологически сортируются между guards

```mc
f(x) =
    result
    where
        x >= 0             // guard A
        y = sqrt(x)        // переменная (зависит от x, которое выше guard A)
        y < 10             // guard B (зависит от y)
        result = y^2       // переменная (зависит от y)
```

Итоговый порядок:
1. Guard A: `if (!(x >= 0)) return NAN`
2. Переменная: `y = sqrt(x)`
3. Guard B: `if (!(y < 10)) return NAN`
4. Переменная: `result = y*y`
5. `return result`

---

## Обнаружение циклов

Если переменные зависят друг от друга по кругу, компилятор выдаёт ошибку:

```mc
f(x) =
    a
    where
        a = b + 1
        b = a - 1   // цикл: a → b → a
```

```
Error E050: circular dependency in where block: a → b → a
```

---

## Диагностика порядка

Для отладки можно использовать `--tokens` чтобы увидеть AST, или добавить промежуточные функции:

```mc
_step1(x) = x^2
_step2(x) = sqrt(_step1(x))

f(x) = _step2(x)
```
