# solve — численное решение уравнений

Находит корень уравнения методом бисекции на заданном интервале.

---

## Синтаксис

```mc
solve(var, lo, hi) { выражение }
```

`solve` ищет значение `var` в диапазоне `[lo, hi]`, при котором `выражение` обращается в ноль.

---

## Примеры

### Решение алгебраического уравнения

```mc
// Найти x: x^3 - x - 2 = 0 на интервале [1, 2]
root(a, b, c) = solve(x, a, b) { x^3 - x - c }
```

### Обратная интерполяция

```mc
// При какой температуре плотность воды равна 999?
temp_for_density(target) = solve(t, 0, 100) { density(t) - target }

density(t) = 1000 - 0.003 * (t - 4)^2
```

### Inline (без фигурных скобок)

```mc
zero_crossing(f) = solve(x, -10, 10) f(x)
```

---

## Генерируемый C-код

`solve(x, lo, hi) { expr }` разворачивается в цикл бисекции из 100 итераций:

```c
mc_num _sol_0 = NAN;
{
  mc_num _lo0 = (lo), _hi0 = (hi), _fl0 = (expr_at_lo);
  for (int _i0 = 0; _i0 < 100; _i0++) {
    mc_num x = (_lo0 + _hi0) * 0.5;
    mc_num _fm0 = (expr_at_mid);
    if (fabs(_fm0) < 1e-9) { _sol_0 = x; break; }
    if (_fl0 * _fm0 < 0.0) _hi0 = x;
    else { _lo0 = x; _fl0 = _fm0; }
  }
  if (isnan(_sol_0)) _sol_0 = (_lo0 + _hi0) * 0.5;
}
```

---

## Точность и поведение

- Точность: `1e-9` — если тело меньше по модулю, результат найден
- Итерации: максимум 100 — если корень не найден с нужной точностью, возвращается середина последнего интервала
- Предусловие: `f(lo)` и `f(hi)` должны иметь разные знаки (теорема о промежуточном значении); иначе бисекция может дать неверный результат

---

## Ограничения

- Один корень на интервал — если на `[lo, hi]` несколько корней, найдёт один из них
- Интервал должен быть конечным (`lo` и `hi` — числа, не `inf`)
