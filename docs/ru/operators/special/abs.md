# Оператор |x| (модуль)

---

## Синтаксис

```mc
|x|
\abs{x}
\lvert x \rvert
abs(x)   // функциональный эквивалент
```

---

## Пример

```mc
distance(a, b) = |a - b|
```

```c
mc_num distance(mc_num a, mc_num b) { return fabs(a - b); }
```

---

## Разбор контекста: `||` vs `|...|`

Лексер различает `||` (логическое ИЛИ) и `|...|` (модуль) по контексту предыдущего токена:

- После числа, идентификатора, `)` — `|` закрывает модуль или начинает `||`
- После оператора, `(`, начала строки — `|` открывает модуль

```mc
f(x, y) = |x| + |y|      // два модуля
g(x, y) = x > 0 || y > 0 // логическое ИЛИ
h(x, y) = |x| || |y|     // модуль, затем ИЛИ, затем модуль
```

---

## Вложенные модули

```mc
f(x, y) = ||x| - |y||   // модуль разности модулей
```

---

## Норма вектора

Для вектора `|v|` вычисляет евклидову норму:

```mc
length(v: num[]) = |v|   // → norm(v) → sqrt(sum(v.*v))
```

---

## Ошибки

### Незакрытый модуль

```mc
f(x) = |x + 1   // ошибка: | не закрыт
```
