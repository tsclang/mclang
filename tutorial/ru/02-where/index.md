# Урок 2 — Блок where

Когда формула становится длинной, промежуточные вычисления можно именовать в блоке `where`. Это делает код читаемым — формула остаётся похожей на учебник, а вычисляемые части выносятся вниз.

---

## 2.1 Базовый синтаксис

Без `where`:

```mc
bmi(weight, height) = weight / height^2
```

С `where` — более длинный пример, формула нормального распределения:

$$f(x) = \frac{1}{\sigma\sqrt{2\pi}} \cdot e^{-\frac{(x-\mu)^2}{2\sigma^2}}$$

```mc
normal_pdf(x, μ, σ) =
    prefactor * exp(-exponent)
    where
        prefactor = 1.0 / (σ * sqrt(2.0 * π))
        exponent  = (x - μ)^2 / (2.0 * σ^2)
```

Переменные `prefactor` и `exponent` существуют только внутри этой функции. Компилятор разберётся с порядком их вычисления сам (топологическая сортировка), поэтому порядок строк в `where` не важен.

---

## 2.2 Зависимые переменные

Одна переменная `where` может использовать другую:

```mc
// Кинетическая энергия через импульс: E = p²/(2m)
kinetic_from_momentum(p, m) =
    energy
    where
        v      = p / m
        energy = 0.5 * m * v^2
```

Компилятор видит, что `energy` зависит от `v`, и вычислит `v` первым — независимо от порядка строк в `where`.

Можно переставить строки местами — результат тот же:

```mc
kinetic_from_momentum(p, m) =
    energy
    where
        energy = 0.5 * m * v^2    // v используется здесь...
        v      = p / m             // ...но вычислится раньше
```

---

## 2.3 Guard-условия (предохранители)

Строка в `where` без знака `=` — это guard (предохранитель). Если условие нарушено, функция вернёт `NaN`:

```mc
safe_div(a, b) =
    a / b
    where
        b != 0

safe_sqrt(x) =
    sqrt(x)
    where
        x >= 0
```

Guard-и проверяются в порядке объявления. Первый нарушенный guard прерывает вычисление.

Реальный пример — синус угла Снелла (угол не должен быть больше π/2):

```mc
snell_angle(n1, θ1, n2) =
    arcsin(ratio)
    where
        ratio = n1 * sin(θ1) / n2
        ratio <= 1.0              // полное внутреннее отражение
        θ1 >= 0.0
        θ1 <= π / 2.0
```

Если `ratio > 1` — физически невозможный угол — функция вернёт `NaN` вместо `arcsin` от числа, большего 1.

---

## 2.4 Повторное использование вычислений

`where` хорошо работает, когда одно подвыражение встречается в формуле несколько раз. Классический пример — дискриминант квадратного уравнения:

$$x_{1,2} = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}$$

```mc
quad_x1(a, b, c) =
    (-b + sqrt(D)) / (2.0 * a)
    where
        D = b^2 - 4.0*a*c
        D >= 0         // нет вещественных корней при D < 0

quad_x2(a, b, c) =
    (-b - sqrt(D)) / (2.0 * a)
    where
        D = b^2 - 4.0*a*c
        D >= 0
```

`D` вычисляется один раз и используется дважды в каждой функции. В сгенерированном Си-коде будет:

```c
mc_num quad_x1(mc_num a, mc_num b, mc_num c) {
    mc_num D = ((b * b) - ((4.0 * a) * c));
    if (!(D >= 0.0)) return NAN;
    return ((-b + sqrt(D)) / (2.0 * a));
}
```

## 2.5 Развёрнутый пример: гармонический осциллятор

Формула положения гармонического осциллятора:

$$x(t) = A \cos(\omega t + \varphi)$$

где угловая частота $\omega = \sqrt{k/m}$ и период $T = 2\pi/\omega$.

```mc
// Позиция осциллятора в момент времени t
oscillator_pos(k, m, A, φ, t) =
    A * cos(ω * t + φ)
    where
        ω = sqrt(k / m)

// Период колебания
oscillator_period(k, m) =
    2.0 * π / ω
    where
        ω = sqrt(k / m)

// Максимальная скорость (в точке равновесия)
oscillator_vmax(k, m, A) =
    A * ω
    where
        ω = sqrt(k / m)

// Максимальная сила (упругая)
oscillator_fmax(k, A) = k * A
```

Проверьте: пружина с `k = 10 Н/м`, масса `m = 1 кг`, амплитуда `A = 0.1 м`:
- `ω ≈ 3.162 рад/с`
- Период `T ≈ 1.987 с`
- Максимальная скорость `≈ 0.316 м/с`

## 2.6 Именованные геометрические расчёты

Площадь треугольника по трём сторонам (формула Герона):

$$S = \sqrt{p(p-a)(p-b)(p-c)}$$

где $p = (a+b+c)/2$ — полупериметр.

```mc
triangle_area(a, b, c) =
    sqrt(p * (p-a) * (p-b) * (p-c))
    where
        p = (a + b + c) / 2.0
        // Треугольное неравенство:
        a + b > c
        b + c > a
        a + c > b
```

Три guard-а гарантируют, что переданные стороны действительно образуют треугольник.

## Задачи

**Задача 1.** Напишите функцию `cylinder_volume(r, h)` через `where`, вычислив сначала площадь основания `base_area = π * r²`.

**Задача 2.** Кинетическая энергия через скорость и угол:

$$E_x = \frac{1}{2} m v^2 \cos^2(\theta)$$
$$E_y = \frac{1}{2} m v^2 \sin^2(\theta)$$

Напишите функции `kinetic_x(m, v, θ)` и `kinetic_y(m, v, θ)`, используя `where` для вычисления `half_mv2 = 0.5 * m * v^2` чтобы не дублировать это выражение.

**Задача 3.** Формула Хаверсина — расстояние по поверхности сферы:

$$d = 2R \arcsin\!\left(\sqrt{\sin^2\!\frac{\Delta\phi}{2} + \cos\phi_1\cos\phi_2\sin^2\!\frac{\Delta\lambda}{2}}\right)$$

Реализуйте `haversine(lat1, lon1, lat2, lon2)` с `R = 6371000.0` через `where`, назвав подвыражения `dlat`, `dlon`, `a`. Вычислите расстояние Москва (55.75°, 37.62°) → Лондон (51.51°, -0.13°) в километрах. Координаты в градусах, но sin/cos принимают радианы — используйте `deg2rad(d) = d * π / 180.0`.

**Задача 4 (★).** Добавьте в `haversine` guard-и:
- Широта должна быть в диапазоне `[-π/2, π/2]`
- Долгота — в `[-π, π]`

Оба параметра — в радианах.

**Задача 1:**

cylinder_volume(r, h) =
    base_area * h

```mc
kinetic_x(m, v, θ) =
    half_mv2 * cos(θ)^2
    where
        half_mv2 = 0.5 * m * v^2

kinetic_y(m, v, θ) =
    half_mv2 * sin(θ)^2
    where
        half_mv2 = 0.5 * m * v^2
```

**Задача 3:**
```mc
R_earth = 6371000.0

deg2rad(d) = d * π / 180.0

haversine(lat1, lon1, lat2, lon2) =
    2.0 * R_earth * arcsin(sqrt(a))
    where
        dlat = lat2 - lat1
        dlon = lon2 - lon1
        a    = sin(dlat/2.0)^2 + cos(lat1)*cos(lat2)*sin(dlon/2.0)^2
```

Для Москвы и Лондона (в радианах): `haversine(0.9730, 0.6566, 0.8988, -0.0023) ≈ 2499 км`.

        lat2 <=  π/2.0

        dlat = lat2 - lat1
