# MClang — Спецификация

Вот подробная фиксация всех проектных решений для твоего нового языка mclang. Этот документ можно использовать как техническое задание (Roadmap) для дальнейшей разработки.

## 1. Концепция и позиционирование

* Название: mclang (от Math C Language).
* Суть: Специализированный язык-вычислитель для математических формул, который компилируется в чистый Си.
* Роль: «Язык-библиотека». Он не предназначен для написания полноценных приложений с интерфейсом и вводом-выводом (I/O). Его задача — принять аргументы, быстро посчитать формулу и вернуть результат.
* Встраиваемость: Дизайн ориентирован на легкую интеграцию в JavaScript (через WebAssembly), Python (через C-extensions) и C/C++ (как нативный модуль).

## 2. Философия проекта

* Цель: Создание быстрых встраиваемых вычислительных библиотек.
* Сфера применения: Геймдев (физика), Data Science, инженерные расчеты.
* Связующее звено: Компилируется в автономный Си-код (без зависимостей), который легко импортируется в JS (Wasm), Python (C API) или C++.

## 3. Синтаксис и Структура

* Декларативный стиль: Функции определяются как математические равенства: `f(x) = выражение`.
* Значимые отступы (Python-style): Код структурируется отступами. Препроцессор вставляет виртуальные `{}` для Си.
* Line Wrapping: Если строка заканчивается на оператор или содержит незакрытую скобку (круглую или фигурную от LaTeX), препроцессор понимает, что выражение продолжается на следующей строке, и не ставит `;`.
* Никаких строк: Язык оперирует только числами. Текстовые данные отсутствуют для обеспечения безопасности и скорости.
* Авто-точка с запятой: Добавит `;` в конце.

Точка с запятой (;) ставится в конце каждой строки, если она содержит законченное выражение, кроме случаев, когда строка является заголовком блока (заканчивается на =, if, else, for, while), директивой #, или имеет явный признак продолжения на следующей строке (заканчивается на оператор или открытую скобку, либо следующая строка начинается с оператора).

Короткий чек-лист препроцессора:

Не ставить `;`, если в конце: `+ - * / , = ( [ {` или слова `if`, `else`, `for`, `while`. Или если в начале следующей строки: `+ - * / . ->`.

Ставить `;` во всех остальных случаях, кроме пустых строк, комментариев и `#include`.

Современные языки позволяют использовать математические символы прямо в коде:

```
π = 3.1415
α = β + γ
```

Препроцессор будет просто заменять их на допустимые в Си названия переменных (например, `alpha`, `beta`).

Си (особенно старые стандарты) плохо переваривает Unicode в названиях переменных. Компилятор должен делать транслитерацию или хеширование Unicode-символов.

Пример: `π` преобразуется в `__uni_pi` или `_u03C0` в итоговом Си-коде. Для пользователя это `π`, для GCC — валидный ASCII.

Нижние индексы: `x_{i}` → `x[i]` (если это массив) или `x_i` (если это часть имени переменной).

### Типы данных

Главная особенность — отсутствие необходимости вручную указывать типы данных для математических расчетов.

* Total Type Inference: По умолчанию все переменные — это универсальное число `num`. Пользователь не пишет `float` или `double`.
* Адаптивная точность: Тип `num` не "прибит" к разрядности. На этапе компиляции он превращается в то, что нужно целевому железу:
    * JS/Python/PC: `double` (64-бит).
    * STM32/ESP32: `float` (32-бит).
    * Arduino/8-bit: `fixed-point` или `float` (через флаги оптимизации).
* Тип `int`: Используется только явно и только для индексов или счетчиков итераций.

Пользователь пишет: `f(x, y) = x + y`.
Компилятор генерирует: `mc_num f(mc_num x, mc_num y) { return x + y; }`.

А в начале заголовочного файла (.h) добавляется умный макрос:

```c
#ifdef MC_USE_FAST_FLOAT
    typedef float mc_num;  // Для 32-битных систем
#elif defined(MC_USE_8BIT)
    typedef int16_t mc_num; // Для совсем слабых систем (фиксированная точка)
#else
    typedef double mc_num;  // Стандарт для PC/JS/Python
#endif
```

Ты запускаешь компилятор и указываешь, где будешь использовать код.

Для веба: `mclang physics.mc --target wasm`
Для Arduino: `mclang physics.mc --target avr --precision f32`
Для Python: `mclang physics.mc --target shared`

Это делает mclang максимально декларативным. Мы уходим от "программирования" в сторону "описания вычислений".

В JavaScript (WebAssembly)

Твой JS-разработчик просто импортирует формулу как обычную функцию.

```javascript
import { trajectory } from './physics.wasm';

// Всё работает быстро, числа передаются как обычные JS-numbers
const height = trajectory(1.5, 20.0, 45.0);
console.log(`Высота: ${height}м`);
```

В Python (Data Science)

Для Python-скрипта это будет выглядеть как нативный модуль, написанный на Си.

```python
import mclang_physics as mc

# Считаем силу удара
force = mc.impact_force(mass=70.5, velocity=15.0, time=0.1)
print(f"Сила: {force}H")
```

В Си (Микроконтроллер/8-бит)

Для прошивки Arduino ты просто подключаешь сгенерированный заголовочный файл.

```c
#include "physics.h"

void loop() {
    // На 8-битном чипе num автоматически стал float или fixed-point
    float h = trajectory(1.2, 10.0, 30.0);
    digitalWrite(LED_PIN, h > 5.0 ? HIGH : LOW);
}
```

### Вектора и матрицы

1. Объявление и Типы

Поскольку у нас только num и int, массивы определяются просто:

* `v: num[]` — вектор (одномерный массив).
* `m: num[][]` — матрица (двумерный массив).

Литералы:

```
v = [1, 2, 3]
m = [[1, 0], [0, 1]]
```

2. Операции «из коробки» (Overloading)

В Си нельзя просто сложить два массива через `+`. В mclang это будет работать как в математике:

* Сложение/Вычитание: `v1 + v2` — поэлементное сложение.
* Умножение на скаляр: `2v` или `v * 2`.
* Матричное умножение: `m1 * m2` (автоматически генерирует в Си вложенные циклы для умножения «строка на столбец»).
* Скалярное произведение: `v1 ⋅ v2` (через точку `\cdot`) или функция `dot(v1, v2)`.

3. Индексация и Слайсы

Индексы начинаются с 0 (как в Си/JS).

* `v[0]` — первый элемент.
* `m[row][col]` — элемент матрицы.
* `v[0..2]` — взять срез (первых три элемента).

4. Специальные функции для матриц

Для физики и химии нам понадобятся:

* `transpose(m)` или `mᵀ` — транспонирование.
* `det(m)` — детерминант.
* `inv(m)` — обратная матрица.
* `norm(v)` — длина вектора.

5. Декларативное создание (Comprehensions)

Это позволит создавать сложные матрицы одной строкой:

```
// Создать единичную матрицу 3x3
I = [[1 if i == j else 0 for i in 0..2] for j in 0..2]
```

6. Трансляция в Си

Здесь есть два варианта, которые компилятор выберет сам:

   1. Статические (Stack): Если размер известен (`v: num[3]`), Си создает обычный массив: `double v[3]`. Это работает молниеносно.
   2. Динамические (Heap): Если размер меняется, mclang генерирует структуру с указателем. Но так как мы договорились минимизировать `malloc`, приоритет будет за статическими размерами.

Пример в mclang:

```
// Расчет кинетической энергии для системы частиц
total_energy(masses: num[], velocities: num[]) =
    ∑(i=0, masses.length - 1) \frac{masses[i] * velocities[i]^2}{2}
```

Итог для парсера:

   1. Добавляем поддержку квадратных скобок [].
   2. Добавляем операторы ⋅ (dot) и ⨯ (cross).
   3. Реализуем свойство .length для массивов.

### Операции с матрицами

Для матриц в mclang мы внедрим синтаксис, который объединяет наглядность MATLAB и строгость Си. Поскольку мы работаем только с числами, компилятор сможет генерировать максимально быстрые «плоские» циклы.

1. Объявление и создание

Мы будем использовать двойные квадратные скобки. Для удобства чтения разрешим перенос строк без запятых (как в Python/Julia).

```
// Матрица 2x3
A = [
    [1, 2, 3]
    [4, 5, 6]
]

// Матрица в одну строку
B = [[1, 0], [0, 1]]
```

2. Базовые арифметические операции

В отличие от Си, в mclang операторы над матрицами работают «из коробки»:

* Поэлементное сложение/вычитание: `A + B` или `A - B`.
* Умножение на число: `2A` или `A * 2`.
* Матричное умножение (Dot product): `A ⋅ B`.
* Примечание: Компилятор автоматически проверит размерности на этапе сборки (если они известны) или добавит проверку в рантайм.
* Поэлементное умножение: `A .* B` (как в MATLAB/Julia), если нужно перемножить элементы, а не матрицы по правилам алгебры.

3. Специальный синтаксис (Sugar)

Для физиков и инженеров добавим поддержку LaTeX-символов:

* Транспонирование: `Aᵀ` (Unicode `\top`) или `A'`. В Си превратится в функцию, меняющую индексы местами.
* Определитель: `det(A)` или `|A|`.
* Обратная матрица: `inv(A)` или `A^{-1}`.

4. Доступ к элементам и Слайсы (Slices)

```
val = A[0][1]    // Элемент в первой строке, втором столбце
row = A[0]       // Первая строка целиком (возвращает вектор)
col = A[:, 1]    // Второй столбец (все строки, индекс 1)
```

5. Трансляция в Си: Оптимизация

Это самое важное. Чтобы не плодить `malloc`, компилятор будет стараться использовать одномерные массивы (Flat Arrays).

Пример в mclang:

```
C = A * B
```

Как это развернется в Си:

```
// Вместо медленных вложенных указателей double**
// mclang генерирует плоский массив double C[rows * cols]
for (int i = 0; i < rows_A; i++) {
    for (int j = 0; j < cols_B; j++) {
        C[i * cols_B + j] = 0;
        for (int k = 0; k < cols_A; k++) {
            C[i * cols_B + j] += A[i * cols_A + k] * B[k * cols_B + j];
        }
    }
}
```

Если целевая архитектура поддерживает SIMD (например, AVX), компилятор может заменить эти циклы на векторные инструкции.

6. Матричные константы

* `I(n)` — Единичная матрица размера `n`.
* `zeros(r, c)` — Матрица, забитая нулями.
* `ones(r, c)` — Матрица из единиц.

## 4. Математический движок

mclang объединяет удобство Julia и мощь LaTeX:

* Неявное умножение: Поддержка записи вида `2x`, `10(a+b)` или `2πr`. Парсер автоматически вставляет оператор \* между числом и переменной/скобкой.
* LaTeX-синтаксис: Прямая поддержка степеней `x^2`, дробей `\frac{a, b}` и корней `\sqrt{x}`.
* Unicode: Можно использовать греческие буквы и спецсимволы (`α`, `β`, `π`, `∑`) в качестве имен переменных. При компиляции они транслитерируются в ASCII-безопасные имена для Си.
* Блок `where`: Возможность выносить уточняющие расчеты под основную формулу для чистоты чтения.
* Цепочки сравнений: `0 < x < 100` транслируется в корректную логику Си.

Почему это удобно?

Математик может просто скопировать формулу из научной статьи или Википедии (где формулы обычно в LaTeX), вставить в .mc файл, и она сразу заработает.

Пример сложной формулы:

```
// Исходник в mclang
y = \frac{1}{\sqrt{2πσ^2}} * e^{-\frac{(x-μ)^2}{2σ^2}}
```

Что выдаст твой компилятор для Си:

```c
double y = (1.0 / sqrt(2 * pi * pow(sigma, 2))) * exp(-(pow(x - mu, 2) / (2 * pow(sigma, 2))));
```

1. Перегрузка операторов (Syntactic Sugar)

В чистом Си нельзя написать `vA = vB + vC` для векторов. Придется вызывать функции. Но в твоем языке ты можешь это реализовать на этапе компиляции:

* Если парсер видит `+` между объектами типа `Vector3`, он транслирует это в вызов функции `vector3_add(vB, vC)`.
* Это сделает математический код на твоем языке чистым, как в Python или GLSL.

2. Типизация: Матрицы и Векторы

Для системного языка важно, чтобы математика была "дружелюбна" к железу. Можно заложить поддержку:

* SIMD-инструкций: При компиляции в Си использовать интринсики (SSE/AVX), чтобы операции над массивами чисел выполнялись параллельно на уровне процессора.
* Статически типизированные размерности: Например, `Matrix<4,4>` — компилятор сразу выделит нужный объем памяти в стеке, как в Си.

3. Структура библиотеки

* Core: Обертки над стандартными функциями Си (`abs`, `sqrt`, `sin`, `pow`), но с поддержкой перегрузки (чтобы `abs()` работал и для `int`, и для `float`, и для векторов).
* Geometry: Векторы (`Vec2`, `Vec3`), Матрицы (`Mat4`), Кватернионы. Это база для графики и физики.
* Fast Math: Приблизительные, но очень быстрые алгоритмы (например, знаменитый "быстрый обратный квадратный корень").

## 5. Синтаксис

### Константы

В mclang работа с константами должна быть такой же естественной, как в справочнике. Мы реализуем их на двух уровнях:

1. Встроенные константы (Стандарт)

Парсер будет нативно узнавать самые популярные математические символы. Тебе не нужно их определять — они просто «есть» в языке:

* `π` или `pi` (3.14159...)
* `e` (2.71828...)
* `τ` (tau = 2π)
* `φ` (phi — золотое сечение)

Встроенные доступны всегда "из коробки". Тебе не нужно писать `π = 3.14...`, компилятор уже это знает.

2. Глобальные константы пользователя

Ты можешь определять свои константы в начале файла. В Си они превратятся либо в `const num`, либо в `#define` (для экономии памяти на 8-битных системах).

```
// Глобальные константы в начале файла
G = 6.67430e-11
c = 299792458

// Использование в формуле
force(m1, m2, r) = G * \frac{m1 * m2, r^2}
```

Если ты пишешь `my_const = 3.14159` на верхнем уровне файла (вне функций), это становится неизменяемым значением.

Его нельзя переопределить внутри функций (кроме как через блок `where`).

Если попытаться написать `π = 4` внутри функции, компилятор выдаст ошибку: «π — это константа». Это защищает от случайных багов (например, если кто-то решит, что в его расчетах «Пи» равно трем).

Но так же можно переопределить встроенную `π` как пользовательскую, если написать ее в начале файла, например:

`π = 3.14`

Это называется переопределением на уровне модуля.

* Гибкость: Если математику для конкретной задачи (например, в инженерных упрощенных расчетах или в рамках какой-то специфической теории) нужно, чтобы `π` было равно или `3.14`, он просто объявляет это один раз в начале своего `.mc` файла.
* Предсказуемость: Это переопределение коснется только этого файла. Если ты импортируешь этот файл в JS, функция будет считать с `π = 3.14`. Но в других файлах `π` останется стандартным.
* Производительность: Компилятор на этапе сборки Си-кода просто подставит `3.14` вместо `3.14159...`, так что на скорости это никак не скажется.

3. Локальные переменные (через where или блоки)

Внутри функции ты создаешь переменные, которые живут только там.

```
circle_area(r) = π * r^2  // Использует глобальную π
```

Можно ли "перекрыть" константу?

Математики иногда используют одни и те же буквы для разных нужд. Мы разрешим локальное перекрытие, если это явно указано в блоке where:

```
// Глобально π — это 3.14...
my_func(x) = x * π
    where
        π = 10  // Внутри этой функции π будет 10, но снаружи останется 3.14
```

Но для системного языка это плохая практика, поэтому mclang будет выдавать Warning (предупреждение), что ты перекрываешь стандартную константу.

4. Интеллектуальное управление константами

Твой компилятор будет делать Constant Folding (свертку констант).

Пример: Если ты написал `2π * r`, компилятор не будет заставлять процессор умножать 2 на 3.14 в рантайме. Он еще на этапе сборки заменит это на готовое число `6.28318....`

Для 8-битных систем это критично: меньше вычислений — выше скорость.

5. Где их хранить?

Чтобы не забивать память на встраиваемых системах:

* Если константа используется один раз — она подставляется числом прямо в формулу.
* Если часто — выносится в секцию readonly памяти.

Итог по константам.

В mclang константа — это просто «имя для числа», которое:

* Понимает Unicode (`π`, `γ`).
* Имеет бесконечную точность до момента компиляции (компилятор сам обрежет её до int, float или double).
* Не требует ключевых слов типа `const` — достаточно просто написать `Имя = Число` вне тела функций.
* Это делает код еще чище. Ты просто описываешь среду (константы), а затем формулы, которые в ней работают.

mclang работает по иерархии приоритетов:

* Локальная переменная (в блоке `where` или внутри функции) — самый высокий приоритет.
* Пользовательская глобальная константа (например `π = 3.14` в начале файла).
* Встроенная константа (дефолтное `3.14159...`) — самый низкий приоритет.

### Дроби

В LaTeX дробь пишется так: `\frac{числитель}{знаменатель}`.

Как это работает в парсере:

* Лексер видит токен `\frac`.
* Парсер ожидает две группы выражений, разделенных запятой.
* Трансляция в Си: Оборачивает их в скобки и ставит деление: `((a) / (b))`.

Пример:

```
y = \frac{x + 1, y - 1} // → ((x + 1) / (y - 1))
```

### Степени

Для степени в mclang мы используем правило «двойного синтаксиса»:

   1. LaTeX-стиль: `x^{n}` или `x^n` — для тех, кто привык к научной нотации.
   2. Программистский стиль: `x**n` — как в Python/Julia.

Оба варианта препроцессор превращает в стандартную функцию Си `pow(основание, степень)`.

Нюансы парсинга:

* Одиночный символ: `x^2` → `pow(x, 2)`.
* Сложное выражение: `x^{a + b}` → `pow(x, (a + b))`.
* Приоритет: Степень имеет самый высокий приоритет. `2x^2` парсится как `2 * (pow(x, 2))`, а не `pow(2x, 2)`.

Пример в коде:

```
f(x) = 2x^2 + e^{x-1}
```

В Си:

```
double f(double x) {
    return 2 * pow(x, 2) + pow(e, (x - 1));
}
```

### Корни

Аналогично LaTeX: `\sqrt{выражение}`.

Как это работает в парсере:

* Видим `\sqrt`.
* Трансляция в Си: Прямая подстановка функции из math.h -> `sqrt(x)`.

В LaTeX корень n-ой степени пишется через квадратные скобки: `\sqrt[n]{x}`.

Как это будет в mclang:

```
// Квадратный корень (обычный)
y = \sqrt{x}            // -> sqrt(x)

// Корень n-ой степени
y = \sqrt[3]{x}         // -> pow(x, 1.0/3.0)
y = \sqrt[n]{x + 1}     // -> pow(x + 1, 1.0/n)
```

### Тригонометрия

* Синтаксис: sin(x), cos(x), tan(x), tg(x).
* Трансляция: Напрямую в sin(), cos(), tan() и т.д.

Важное правило для парсера:

Если функция начинается с `\` (как `\sin`), парсер игнорирует слеш и ищет совпадение в списке стандартных функций Си. Если слеша нет (`sin`), логика та же. Это позволяет писать и `sin(x)`, и `\sin{x}`.

В классическом LaTeX команды `\tg` нет, там используется стандартная `\tan`. Однако в русской математической традиции (и в пакетах типа babel) команда `\tg` и `\ctg` — это стандарт.

Мы сделаем поддержку обоих вариантов:

Тангенс: `\tan{x}` и `\tg{x}` - оба превратятся в `tan(x)`.
Котангенс: `\cot{x}` и `\ctg{x}` - оба превратятся в `(1.0 / tan(x))`.

Для mclang мы добавим расширенную таблицу синонимов, чтобы русская математическая традиция (запись через `tg`, `ctg`, `sh`, `ch`) работала «из коробки» наравне с международными стандартами.

Расширенная таблица тригонометрии mclang:

| Функция | Русская традиция | Международный стандарт / LaTeX | Трансляция в Си (math.h) |
|---|---|---|---|
| Тангенс | tg, \tg | tan, \tan | tan(x) |
| Котангенс | ctg, \ctg | cot, \cot | (1.0 / tan(x)) |
| Арксинус | arcsin, \arcsin | asin | asin(x) |
| Арккосинус | arccos, \arccos | acos | acos(x) |
| Арктангенс | arctg, \arctg | atan, arctan, \arctan | atan(x) |
| Арккотангенс | arcctg, \arcctg | acot, arccot, \arccot | (M_PI_2 - atan(x)) |
| Гиперб. синус | sh, \sh | sinh, \sinh | sinh(x) |
| Гиперб. косинус | ch, \ch | cosh, \cosh | cosh(x) |
| Гиперб. тангенс | th, \th | tanh, \tanh | tanh(x) |
| Гиперб. котангенс | cth, \cth | coth, \coth | (1.0 / tanh(x)) |

Мы создадим карту маппинга (Dictionary). Лексер, встречая любой из этих идентификаторов (со слешем или без), просто заменяет его на ключ, соответствующий функции в Си.

Пример обработки:

```
// mclang
y = \tg{x} + cth(x)
```

   1. `\tg` → находит в таблице `tan`
   2. `cth` → находит в таблице `(1.0 / tanh)`
   3. Итог в Си: `double y = tan(x) + (1.0 / tanh(x))`;

Разработчику не нужно думать, какую библиотеку подключать или как правильно пишется гиперболический котангенс в Си. Он пишет cth(x), а mclang сам генерирует математически верную конструкцию.

### Логарифмы

Здесь мы внедряем удобный LaTeX/Julia синтаксис:

* Натуральный логарифм: `ln(x)` или `log(x)` - `log(x)` в Си.
* Десятичный: `log10(x)` или `\lg{x}` - `log10(x)` в Си.
* Произвольное основание: `log{base}{x}` или `\log_{base}{x}` - в Си транслируется через формулу смены основания: `(log(x) / log(base))`.

Как это выглядит в коде mclang:

```
// Пример сложной функции
f(x, a) = 
    log{a}{x} + \sin{πx} + ln(e^2)
```

Как это видит препроцессор для Си:

   1. `log{a}{x}` - `(log(x) / log(a))`
   2. `\sin{πx}` - `sin(3.14159 * x)`
   3. `ln(e^2)` - `log(pow(2.71828, 2))`

Итог (Си):

```
double f(double x, double a) {
    return (log(x) / log(a)) + sin(3.14159 * x) + log(pow(2.71828, 2));
}
```

### Экспонента и прочее

* Экспонента: `e^x` или `exp(x)` - `exp(x)`.
* Модуль: `abs(x)` - `fabs(x)` для `num`.
* Округление: `floor(x)`, `ceil(x)`, `round(x)`.

### Синтаксис модуля

* LaTeX-стиль: |x| или \abs{x}.
* Программистский стиль: abs(x).

Поскольку наш основной тип — это `num` (который в Си становится `double`), препроцессор всегда транслирует модуль в функцию `fabs()` из библиотеки `math.h` (она работает с числами с плавающей точкой, в отличие от простого `abs()`, который в Си предназначен для целых).

* `|x|` - `fabs(x)`
* `\abs{a + b}` - `fabs(a + b)`

Парсер будет корректно обрабатывать вложенные модули и сложные выражения внутри:

```
// mclang
f(x, y) = |x - |y|| + \abs{\frac{x}{y}}
```

Трансляция в Си:

```
double f(double x, double y) {
    return fabs(x - fabs(y)) + fabs(x / y);
}
```

Для лексера символ `|` может быть коварен (он же используется для логического "ИЛИ" `||`).

Правило:

* Если символ `|` стоит один и за ним идет число/переменная — это начало модуля.
* Если за ним сразу идет второй такой же символ `||` — это логический оператор.

Парсер должен уметь отличать `||` (логическое ИЛИ) от `||` (двойного модуля):

```
f(x, y) = ||x| - |y||
```

```
f(x, y) = x||y
```

### Операции сравнения

В mclang операции сравнения и возврат значений работают на стыке математической логики и системной производительности Си. Поскольку у нас «чистый вычислитель», каждое сравнение должно возвращать число.

Мы поддерживаем стандартный набор, включая удобные математические символы:

| Операция | Синтаксис mclang | Трансляция в Си | Результат |
|---|---|---|---|
| Равно | == | == | 1.0 (true) или 0.0 (false) |
| Не равно | !=, <> | != | 1.0 или 0.0 |
| Больше / Меньше | >, < | >, < | 1.0 или 0.0 |
| Больше или равно | >=, ≥ | >= | 1.0 или 0.0 |
| Меньше или равно | <=, ≤ | <= | 1.0 или 0.0 |

1. Цепочки сравнений (Julia-style)

Это одна из главных фишек.

Вместо громоздкого `x > 0 && x < 10`, ты пишешь как в тетради:

```
is_valid(x) = 0 < x < 10
```

Как это делает препроцессор:

Он разворачивает это в Си-совместимую логику: ```(0 < x && x < 10)```.

2. Возврат значений (Implicit Return).

Как мы уже зафиксировали, ключевое слово return не нужно. Функцией возвращается последнее вычисленное выражение.

Пример с логикой:

```
step_function(x) =
    if x < 0
        0
    else if 0 ≤ x < 10
        x^2
    else
        100
```

Трансляция в Си:

```
double step_function(double x) {
    if (x < 0) return 0;
    if (0 <= x && x < 10) return pow(x, 2);
    return 100;
}
```

3. Логические операторы как числа

В mclang результат сравнения — это число (num). Это позволяет делать крутые математические трюки без if:

```
// Функция вернет x, только если x положительный, иначе 0
// (без использования if)
rectified(x) = x * (x > 0)
```

В Си это будет: `return x * (x > 0);`. Если `x > 0` истинно (1), получим `x * 1`. Если ложно (0) — `x * 0`.

4. Приоритет операций

Сравнения имеют более низкий приоритет, чем арифметика.

`x + 1 > y * 2` парсится как `(x + 1) > (y * 2)`.

Итог по правилам:

   1. Сравнение всегда возвращает `1.0` или `0.0`.
   2. Последняя строка — это результат.
   3. Цепочки `a < x < b` разрешены и поощряются.

### Сложные методы

Мы подходим к границе между аналитическим (символьным) и численным решением.

Например, вычисление пределов, производных или интегралов.

Так как mclang компилируется в Си, у нас есть два пути реализации:

1. Численный подход (Numerical Limits)

Это самый простой и эффективный способ для встраиваемого движка. Предел `\lim_{x \to a} f(x)` вычисляется как значение функции в точке, очень близкой к `a`.

Синтаксис mclang:

```
// Предел функции sin(x)/x при x стремящемся к 0
val = \lim{x \to 0} \frac{sin(x)}{x}
```

Трансляция в Си:

Компилятор генерирует вспомогательный код, который вычисляет значение функции, например, в точке `0 + 1e-9`.

```c
// Внутренняя реализация препроцессором
double x_target = 0.0 + 1e-9; double res = sin(x_target) / x_target;
```

2. Аналитический подход (Compile-time)

Если формула простая, компилятор mclang может попытаться раскрыть неопределенность на этапе сборки (используя правило Лопиталя или упрощение выражений).

* Пример: `\lim{x \to \infty} \frac{1}{x}` компилятор сразу заменит на `0.0` в итоговом Си-коде.

3. Производные и Интегралы

Раз мы заговорили о пределах, давай добавим и их, так как они работают по той же логике:

* Производная: `\frac{d}{dx} f(x)` — вычисляется численно через разностную схему: `(f(x+h) - f(x))/h`.
* Интеграл: `\int{a}{b} f(x) dx` — разворачивается в цикл Си (метод трапеций или Симпсона).

Пример в mclang:

```
// Определенный интеграл от 0 до π
area = \int{0}{π} sin(x) dx
```

Почему это сложно для Си?

Для вычисления интеграла или предела Си-коду нужно уметь «вызывать» переданную формулу много раз.

Решение mclang:

Компилятор будет генерировать внутри Си-файла статическую функцию для этого выражения и передавать указатель на неё в библиотечный метод вычисления (например, `mc_integrate(my_func_ptr, a, b)`).

Итог по сложным методам:

Для mclang мы выберем численные методы (Numerical Methods), потому что они:

   1. Всегда дают результат для любых функций.
   2. Не требуют гигантского движка символьной математики внутри компилятора.
   3. Работают со скоростью Си.

### Бесконечность

В mclang, как и в современном Си (IEEE 754), мы не «падаем» при встрече с бесконечностью или делением на ноль. Вместо этого мы используем специальные числовые состояния.

Для системного языка-библиотеки это самый надежный путь:

1. Бесконечность (`inf`)

В mclang зарезервировано ключевое слово `inf` (или Unicode `∞`).

* `1 / 0` даст `inf`.
* `-1 / 0` даст `-inf`.
* Любое число, превышающее лимит `num` (например, результат очень большой степени), станет `inf`.

В Си это транслируется в: `INFINITY` (из `math.h`).

2. Не-число (`nan`)

Если результат математически не определен, мы возвращаем `nan` (Not a Number).

* `0 / 0` → `nan`.
* `inf - inf` → `nan`.
* `sqrt(-1)` → `nan`.

В Си это транслируется в: `NAN`.

3. Проверка состояний

Раз у нас декларативный стиль, пользователю нужны инструменты, чтобы обрабатывать такие случаи прямо в формулах:

```
f(x, y) = 
    if is_nan(res)
        0
    else
        res
    where
        res = x / y
```

4. Деление на ноль: как это работает в рантайме

Поскольку мы компилируем в Си, поведение будет следующим:

   1. На PC (JS/Python): Программа не вылетит. Ты просто получишь `inf` или `nan` как результат функции и сможешь обработать его в основном коде.
   2. На микроконтроллерах (8-бит): Если чип не поддерживает IEEE 754 аппаратно, компилятор mclang добавит программную проверку (guard), чтобы возвращать заранее определенное «безопасное» значение или флаг ошибки.

5. Пределы и бесконечность

Твой синтаксис LaTeX будет это поддерживать:

```
// Предел при x стремящемся к бесконечности
L = \lim{x \to \infty} \frac{1}{x}  // Вернет 0.0
```

Итог:

* mclang не боится деления на ноль.
* Результатом может быть число, inf или nan.
* Это позволяет строить цепочки вычислений, которые не прерываются из-за одной математической ошибки.

## 6. Операторы

### Управляющие конструкции

* Неявный `return`: Ключевое слово `return` не требуется. Результатом функции становится последнее вычисленное выражение.
* Функциональный `if/else`:
    * Инлайновый: `val = a if cond else b`.
    * Блочный: Через отступы для сложной логики.
* Отсутствие I/O: В языке нет функций `print()` или `input()`. Взаимодействие идет только через аргументы и возвращаемые значения.

1. if/else

Чистый mclang:

```
check(x) = 
    if x < 0
        -1
    sqrt(x)
```

Если после условия if (...) сразу на этой же строке идёт значение, компилятор трактует это как мгновенный возврат или результат выражения.

```
// Максимально короткая запись
check(x) = if (x < 0) -1 else sqrt(x)
```

Это превращается в Си либо в тернарный оператор `(x < 0) ? -1 : sqrt(x)`, либо в классический `if/else`.

Или даже без else (как guard clause):

```
check(x) = 
    if (x < 0) -1
    sqrt(x)
```

Блочный вариант (Для длинных формул)

```
abs_complex(x) =
    if (x < 0)
        -x * \frac{π, 2}
    else
        x * \frac{π, 2}
```

Препроцессор просто проверяет:

* Вижу `if` (условие).
* Смотрю, есть ли что-то на этой же строке после закрывающей скобки `)`.
* Если есть — это однострочный блок.
* Действие: Оборачиваем это в `return ...;` (если это последняя строка функции) или в `if (...) { return ...; }`.

```
// Декларация функции знака числа
sgn(x) = 
    if (x > 0) 1
    if (x < 0) -1
    0  // если ни один if не сработал
```

Трансляция в Си:

```c
double sgn(double x) {
    if (x > 0) return 1;
    if (x < 0) return -1;
    return 0;
}
```

### Циклы (for/while)

Они нужны для итерационных методов и работы с массивами данных.

Численные методы: Многие математические задачи (интегралы, поиск корней, ряды Тейлора) решаются только циклами.

Векторы: Если ты прокидываешь в функцию массив (например, координаты 1000 точек), тебе нужен цикл, чтобы обработать их все за один вызов, не возвращаясь в медленный Python/JS на каждой итерации.

```
// mclang
for i in 1..10
    // i автоматически увеличивается сам
    result = result + i
```

В Си это скомпилируется как:

```c
for (int i = 1; i <= 10; i++) {
    result = result + i;
}
```

Поскольку mclang — это декларативный математический язык, циклы в нём должны выглядеть не как «инструкции процессору», а как описание итерируемых процессов (ряды, суммы, произведения).

Вот как мы реализуем циклы, чтобы они были мощными, но не портили «чистоту» формул:

1. Декларативный цикл `for..in` (Диапазоны)

Это основной вид цикла. Он автоматически преобразуется в эффективный for в Си.

```
// Сумма квадратов от 1 до n
sum_sq(n: int) = 
    res = 0
    for i in 1..n
        res = res + i^2
    res
```

* Синтаксис: `for переменная in начало..конец`.
* Трансляция в Си: `for (int i = начало; i <= конец; i++)`.

2. Шаг итерации (Step)

Если нужно идти не по порядку (например, только по чётным):

```
for i in 0..100 step 2
    // i примет значения 0, 2, 4...
```

* В Си: `for (int i = 0; i <= 100; i += 2)`.

3. Функциональные циклы (`Σ` и `Π`)

Для математика циклы for часто избыточны. Мы введём прямую поддержку знаков суммы и произведения:

* Сумма (`∑`): `\sum_{i=1}^{n} i^2`
* Произведение (`∏`): `\prod_{i=1}^{n} i`

Пример:

```
fact(n) = ∏{i=1}{n} i
```

Компилятор сам развернет это в цикл `for` с переменной-аккумулятором.

4. Цикл `while` (Условный)

Нужен для итерационных методов (например, поиск корня методом Ньютона), где мы не знаем заранее количество шагов.

```
// Поиск корня через итерации
sqrt_newton(val) =
    x = val / 2
    while |x^2 - val| > 0.001
        x = (x + val / x) / 2
    x
```

5. Важные правила для циклов в mclang:

   1. Результат цикла: Последнее вычисленное значение внутри цикла не возвращается автоматически. Чтобы вернуть результат, нужно явно указать переменную после цикла (как в примере `res` выше).
   2. Отсутствие мутабельности снаружи: Мы разрешаем менять переменные внутри функции (`res = res + i`), но эти изменения не выходят за пределы вызова функции.
   3. Исключение `break` / `continue`: Мы их не добавляем. Для выхода из цикла используй условия внутри `while` или `if`. Это делает логику циклов линейной и предсказуемой для Си-оптимизатора.

Итог по циклам:

Циклы в mclang — это способ превратить многократные вычисления в один компактный блок. Для простых сумм юзер использует `∑`, для сложных алгоритмов — `while` или `for`.

### Локальные определения через `where`

Это «киллер-фича» для декларативного стиля. Ты сначала пишешь главную формулу, а ниже расшифровываешь её компоненты. Это позволяет коду читаться как текст: «Результат равен A + B, где A — это..., а B — это...».

```
trajectory(v0, angle, t) = v0 * t * cos(angle) - \frac(gt^2, 2)
    where
        g = 9.81
        angle_rad = angle * \frac(π, 180)
```

### Операторы как в математике

Суммирование (`∑`):

`sum(n) = ∑(i=1, n) i^2` — это автоматически развернется в цикл `for` в Си.

Принадлежность (`∈`):

`check(x) = x ∈ [0, 1]` — развернется в `(x >= 0 && x <= 1)`.

Как это будет выглядеть в итоге (Пример файла .mc):

```
// Определение константы
G = 6.674e-11

// Декларативная функция
force(m1, m2, r) = G * \frac(m1 * m2, r^2)
    where
        m1 > 0
        m2 > 0

// Сложная функция с условиями
energy(m, v, h) = 
    E_kin + E_pot
    where
        E_kin = \frac(mv^2, 2)
        E_pot = mgh
        g = 9.81
```

### Принадлежность

В математике это база, а для компилятора это станет мощным инструментом валидации входных данных.

Используем классический символ принадлежности `∈` (`\in` в LaTeX) или ключевое слово `in`.

В mclang это будет работать как логический оператор, возвращающий 1.0 (истина) или 0.0 (ложь).

* К стандартным множествам:
    * x ∈ \mathbb{N} — Натуральные числа (1, 2, 3...).
    * x ∈ \mathbb{Z} — Целые числа (...-1, 0, 1...).
    * x ∈ \mathbb{R} — Вещественные числа (всегда true для num).
* К диапазонам:
    * x ∈ [0, 1] — Отрезок (включая границы).
    * x ∈ (0, 1) — Интервал (исключая границы).

Для отрицания принадлежности мы будем использовать либо перечеркнутый символ `∉` (`\notin` в LaTeX), либо конструкцию `not in` / `!in`.

* LaTeX / Unicode: `x ∉ ℕ` (символ `\notin`).
* Программистский: `x !in ℕ` или `not x in ℕ`.

Препроцессор просто оборачивает обычную проверку на принадлежность в отрицание !.

Пример: `x ∉ [0, 1]`

В Си: `!(x >= 0 && x <= 1)`

Это идеально подходит для «защиты» формул (Guard Clauses):

```
// Факториал работает только для натуральных чисел
fact(n) = 
    if n ∉ \mathbb{N}
        nan
    else
        // логика расчета
```

Компилятор превращает эти проверки в эффективные логические выражения:

* Натуральные (`\mathbb{N}`): `(n > 0 && floor(n) == n)`
* Целые (`\mathbb{Z}`): `(floor(n) == n)`
* Диапазон `[a, b]`: `(x >= a && x <= b)`

Мы зарезервируем Unicode-символы двойных букв (Double-struck) для стандартных множеств:

* `ℕ` (`\mathbb{N}`)
* `ℤ` (`\mathbb{Z}`, `2ℤ` — это сокращенная запись для множества четных чисел, `2ℤ + 1` — для множества нечетных)
* `ℝ` (`\mathbb{R}`)
* `ℚ` (`\mathbb{Q}` — рациональные)

Пример использования:

```
// Проверка: x — четное натуральное число?
is_even_nat(x) = (x ∈ ℕ) && (x % 2 == 0)
```

### Остаток от деления

Для остатка от деления мы будем использовать стандартный оператор `%`, но с учетом того, что наш основной тип — `num` (число с плавающей точкой), трансляция в Си будет «умной».

1. Синтаксис

* Программистский: `x % y`
* Функциональный: `mod(x, y)`

2. Трансляция в Си

В Си оператор `%` работает только с целыми числами (`int`). Для чисел с плавающей точкой (`double`) используется функция `fmod()`.

* Если оба операнда — целые (`int`): `x % y`
* Если хотя бы один — дробный (`num`): `fmod(x, y)`

3. Применение к множествам (Четность/Нечетность)

Как мы обсуждали ранее, проверка на четность в mclang выглядит очень изящно:

* Четные: `x % 2 == 0` (или `x ∈ 2ℤ`)
* Нечетные: `x % 2 != 0` (или `x ∈ 2ℤ + 1`)

Пример в коде:

```
is_odd(n: int) = n % 2 != 0
```

4. Отрицательные числа

mclang будет следовать поведению Си (`fmod`): знак результата будет таким же, как у делимого (`x`).

### Отрицание

В mclang мы объединяем математическую и программную нотацию. Отрицание будет работать во всех слоях: от простых логических выражений до проверки принадлежности к множествам.

Для инверсии любого логического значения используем либо математический символ `¬`, либо привычные `not` / `!`.

Синтаксис: `¬(x > 0)`, `not (x == 0)`, `!(x < y)`.
Трансляция в Си: Все варианты превращаются в префиксный `!`.

Благодаря правилу «однострочников», в условиях (`if`) можно писать очень лаконичные проверки:

```
// Если x НЕ четное — вернуть 0
f(x) = 
    if x % 2 != 0
        0
    x / 2
```

Отрицание (`not`, `!`, `¬`) имеет очень высокий приоритет (как и в Си/JS). Оно применяется к ближайшему операнду:

`!x > 0` будет понято как `(!x) > 0` (что почти всегда вернет `0.0`, так как `!x` станет `1.0` или `0.0`).

Поэтому для формул рекомендуем скобки: `!(x > 0)`.

### Булевы операции

Для mclang мы выберем синтаксис, который будет понятен и программисту, и математику. Поскольку мы транслируем всё в Си, логика будет работать по правилам: 0 — ложь, всё остальное — истина.

| Операция | Синтаксис mclang | Си-эквивалент | Примечание |
|---|---|---|---|
| И (AND) | `and`, `&&`, `∧` | `&&` | 1.0 если оба операнда не 0 |
| ИЛИ (OR) | `or`, `||` , `∨` | `||` | 1.0 если один операнд не 0 |
| Исключающее ИЛИ (XOR) | `xor`, `⊕` | (a != 0) ^ (b != 0) | В Си `^` побитовый, мы сделаем логический |

Так как в Си оператор `^` работает с целыми битами, для наших чисел `num` (`float`/`double`) компилятор будет генерировать безопасную проверку:

```
// mclang:
a xor b
```

```
// Си результат:
((a != 0.0) ^ (b != 0.0)) ? 1.0 : 0.0;
```

Как и в Си, операторы `and` и `or` будут "ленивыми":

* В `A and B`, если `A` — ложь, `B` вообще не будет вычисляться.
* Это позволяет писать безопасные формулы: `if (x != 0 and 1/x > 0.5)`.

```
// Математический вид с использованием Юникода
check(x, y) = (x > 0 ∧ y > 0) ∨ (x ⊕ y)

// Программистский вид
check(x, y) = (x > 0 and y > 0) or (x xor y)
```

Битовых операций (Bitwise) не будет.

## 7. Физический движок

Для того чтобы mclang стал полноценным инструментом для физиков и химиков, нам не хватает трех фундаментальных вещей: размерностей, индексации (для частиц/состояний) и константной базы.

1. Поддержка размерностей (Units)

В физике число без единиц измерения — это риск (вспомни аварию Mars Climate Orbiter). Мы можем внедрить «синтаксический сахар» для размерностей, который на этапе компиляции будет проверять совместимость, но в Си превратится в обычные числа.

```
// Пример физического расчета
gravity(m1: kg, m2: kg, r: m) =
    G * \frac{m1 * m2}{r^2}
```

* Зачем: Если пользователь попытается сложить `10kg + 5m`, компилятор выдаст ошибку еще до генерации Си-кода.

2. Нижние индексы (Subscripts)

В химии и физике индексы повсюду: `H_2O`, `v_{init}`, `E_k`. В mclang мы можем реализовать их через LaTeX-стиль или подчеркивание.

* Химические формулы: `H_2O` или `H_{2}O`.
* Состояния: `v_{start}`, `v_{end}`.
* В Си: Это превратится в части имени переменной (`v_start`) или в элементы массива, если это база данных частиц.

3. Научные константы «из коробки»

Математик знает `\pi`, но физику нужны:

* `c` (скорость света)
* `G` (гравитационная постоянная)
* `h` (постоянная Планка)
* `N_A` (число Авогадро)
* `R` (универсальная газовая постоянная)

Их нужно зашить в стандартную библиотеку mclang, чтобы не заставлять юзера гуглить их точность.

4. Работа с массивами и тензорами

В физике мы постоянно работаем с векторами сил и тензорами напряжений.

```
// Сила как вектор
F = m * a // где a и F — векторы [x, y, z]
```

Нам нужно добавить в mclang встроенную поддержку векторных операций, чтобы `a * b` для векторов автоматически превращалось в скалярное или векторное произведение (в зависимости от оператора).

5. Спец-функции (Special Functions)

Для квантовой физики или термодинамики обычного `sin`/`cos` мало. Нужны:

* `erf(x)` (функция ошибок)
* `gamma(x)` (гамма-функция)
* Бесселевы функции.
* В Си: Они почти все есть в расширенном `math.h` (C99 и выше), их просто нужно прокинуть в наш синтаксис.

   1. Научные префиксы: `10k`, `5m`, `2u` (кило, милли, микро) — автоматическое умножение на `10^n`.
   2. Векторный тип: Чтобы можно было писать `v1 + v2` для массивов из 3-х чисел.
   3. Нижние индексы: `x_i` для красоты и структуры.

## 8. Работа препроцессора (Roadmap для парсера)

   1. Scanner: Распознает `Indent`/`Dedent` и генерирует токены блоков.
   2. Math-Parser: Находит `2x` и вставляет токен умножения; находит `x^2` и превращает в `pow()`.
   3. ASI (Auto Semicolon): Ставит `;` в конце каждой линии, кроме заголовков функций и условий.
   4. C-Generator: Выдает чистый `.c` и `.h` файлы с функцией `double ballistics(double t, double v0, double angle)`.

mclang — это мост между «тетрадной» математикой и системной производительностью.

### Табличные данные

Для mclang работа с табличными данными — это мост между «чистой математикой» и реальными данными из химии, физики или инженерии. Мы реализуем это через статические структуры и интерполяцию.

1. Объявление таблиц

Пользователь может описывать таблицы констант прямо в коде. Это будет выглядеть как именованный массив пар или строк.

```
// Таблица плотности металлов (кг/м³)
density_table = [
    "Al": 2700,
    "Fe": 7874,
    "Au": 19300
]
```

В Си это превратится в эффективную struct или хеш-таблицу (если ключи — строки).

2. Экспериментальные данные (Lookup Tables)

Это важнейшая фича для инженеров — когда функция задана не формулой, а набором точек из эксперимента.

```
// Зависимость давления от температуры (T, P)
steam_data = [
    (100, 101.3),
    (120, 198.5),
    (150, 475.8)
]

// Получение значения с автоматической интерполяцией
p = steam_data(135)
```

Как это работает:

Если ты вызываешь таблицу как функцию `steam_data(135)`, mclang автоматически генерирует в Си алгоритм линейной интерполяции между ближайшими точками `(120 и 150)`.

3. Работа с CSV/JSON (Compile-time)

Чтобы не вбивать данные руками, препроцессор сможет подтягивать внешние файлы:

```
// Подгружаем таблицу Менделеева при компиляции
elements = import_table("./elements.csv")
atomic_mass = elements["Au"].mass
```

На этапе компиляции данные из CSV станут статическим массивом в Си-коде.

4. Многомерные таблицы (Тензоры данных)

Поддержка таблиц с несколькими входными параметрами (например, теплоемкость зависит и от давления, и от температуры):

```
val = heat_map(p, t)
```

Транслируется в билинейную интерполяцию в Си.

5. Поиск в таблицах (Reverse Lookup)

```
// Найти температуру, при которой давление равно 300
target_t = solve(t) { steam_data(t) == 300 }
```

Здесь mclang задействует решатель, который будет бегать по таблице данных.

### Встроенные табличные данные

Для mclang встроенные табличные данные — это не просто списки, а статические справочники, зашитые прямо в бинарный код. Это позволяет вызывать физические или химические параметры так же легко, как число $\pi$, без лишних аллокаций памяти.

Вот как мы это реализуем:

1. Стандартные научные библиотеки (Built-in Datasets)

Язык будет поставляться с набором встроенных «пространств имен» для разных наук. Тебе не нужно их описывать, достаточно импортировать нужную область:

```
import std.chemistry
import std.physics

// Использование: Периодическая таблица
mass = std.chemistry.elements["Au"].atomic_mass

// Использование: Параметры планет
g_mars = std.physics.planets["Mars"].g
```

2. Структура данных в Си

При компиляции эти таблицы превращаются в максимально оптимизированные константные структуры (POD - Plain Old Data):

```
// В итоговом Си-коде
static const Element periodic_table[] = {
    {79, "Au", 196.967, ...},
    // ...
};
```

Это гарантирует нулевую стоимость (zero-cost) обращения: процессор просто читает значение из памяти программ.

3. Интеллектуальный поиск (Compile-time Lookup)

Если ты используешь константный ключ, компилятор mclang подставит значение еще на этапе сборки:

* mclang: `x = elements["H"].mass * 2`
* Си: `double x = 1.008 * 2;` (самой таблицы в рантайме даже не будет в коде, если она больше не используется).

4. Доступные группы данных «из коробки»

Мы заложим в стандарт:

* Chemistry: Атомные массы, температуры кипения/плавления, электроотрицательность.
* Physics: Магнитные постоянные, параметры элементарных частиц, стандартная атмосфера (ISA).
* Math: Таблицы простых чисел (до определенного предела), значения специальных функций (для быстрой интерполяции).

5. Синтаксис «Живой таблицы»

Если данные в таблице представляют собой функцию (например, плотность воды от температуры), mclang разрешит обращаться к таблице как к функции:

```
// Встроенная таблица свойств воды автоматически интерполирует значения
d = std.physics.water_density(22.5) // Вернет плотность между 22 и 23 градусами
```

Итог:

   1. std-модули: Это предкомпилированные .mc (или даже JSON) файлы, которые поставляются с компилятором.
   2. Static Linking: Данные включаются в финальный код только если они реально используются (Tree Shaking).

### Табличные данные для дискриминантов, интегралов, производных, первообразных

Для mclang мы заложим архитектуру CAS (Computer Algebra System) прямо в компилятор. Это значит, что перед тем как генерировать «тяжелый» код, компилятор сверяется с внутренней аналитической базой данных.

Вот как эти таблицы будут работать в движке:

1. Таблица аналитических интегралов и первообразных

Это «мозги» компилятора. Если он видит `\int f(x) dx`, он не запускает цикл, а делает подстановку.

| Выражение в mclang | Первообразная (в Си) | Условие |
|---|---|---|
| `x^n` | `pow(x, n+1) / (n+1)` |` n != -1` |
| `1/x` | `log(abs(x))` | `x != 0` |
| `e^x` | `exp(x)` | - |
| `\sin{x}` | `-cos(x)` | - |
| `\cos{x}` | `sin(x)` | - |
| `\frac{1}{x^2 + a^2}` | `(1/a) * atan(x/a)` | - |

Как это работает: Если в коде стоит `\int_{0}^{1} x^2 dx`, компилятор не считает сумму, он просто выдает: `(pow(1, 3)/3) - (pow(0, 3)/3)`. Это дает бесконечную точность и мгновенную скорость.

2. Таблица производных (Symbolic Differentiation)

Компилятор умеет брать производную от любой композиции базовых функций, используя таблицу правил.

* `\frac{d}{dx} \sin{x}` - `cos(x)`
* `\frac{d}{dx} \ln{x}` - `1/x`
* `\frac{d}{dx} (f \cdot g)` - `f'*g + f*g'` (развернется в Си-код)

3. Таблица дискриминантов и типов уравнений

Для блока solve компилятор использует таблицу «шаблонов», чтобы понять, как решать задачу оптимально.

| Шаблон (AST) | Метод решения | Формула |
|---|---|---|
| `ax + b = 0` | Линейный | `-b / a` |
| `ax^2 + bx + c = 0` | Квадратный | `(-b ± \sqrt{D}) / 2a` |
| `ax^4 + bx^2 + c = 0` | Биквадратный | Замена `$t = x^2$` |
| `a^x = b` | Логарифмический | `log(b) / log(a)` |

4. Где хранятся эти таблицы?

Для тебя как для разработчика это будут JSON-файлы или JS-объекты внутри компилятора:

   1. Pattern Matcher: Ищет совпадение структуры формулы с таблицей.
   2. Transformer: Если совпадение найдено, заменяет узел в дереве (AST) на «готовый ответ».
   3. Fallback: Если в таблице нет такого интеграла или уравнения, подключается «тяжелая артиллерия» — численные методы в рантайме.

5. Пример: «Умный» компилятор

Ввод mclang:

```
res = \int_{0}^{π} \sin{x} dx
```

Процесс:

   1. Нашел в таблице первообразных: `$\int \sin = -\cos$`.
   2. Выполнил подстановку границ.
   3. Упростил: `$-\cos(\pi) - (-\cos(0)) = 1 + 1 = 2$`.

Результат в Си:

```
double res = 2.0; // Компилятор даже не добавил math.h, он просто вычислил константу!
```

### Математические формулы

Для mclang мы зафиксируем библиотеку математических формул как набор аналитических правил. Это позволит компилятору не просто транслировать код, а понимать математику, упрощая её до генерации Си-кода.

1. Алгебраические тождества (Упрощение)

Компилятор использует таблицу тождеств для сокращения вычислений на этапе построения AST:

* Разность квадратов: `a^2 - b^2` → `(a - b)(a + b)` (если это помогает сокращению с дробью).
* Квадрат суммы: `(a + b)^2` → `a^2 + 2ab + b^2`.
* Логарифмические правила: `log(a) + log(b)` → `log(a * b), n * log(a)` → `log(a^n)`.

2. Тригонометрические формулы

Вместо вызова тяжелых функций Си, препроцессор может упрощать выражения:

* Основное тождество: `sin(x)^2 + cos(x)^2` → `1.0`.
* Двойной угол: `sin(2x)` → `2 * sin(x) * cos(x)`.
* Формулы приведения: `sin(x + π/2)` → `cos(x)`.

3. Специальные математические последовательности

Для итерационных расчетов мы вшиваем «генераторы»:

* Арифметическая прогрессия: `∑(i=0, n) (a + i*d)` → Аналитическая формула `(n+1)*(2a + n*d)/2`.
* Геометрическая прогрессия: `∑(i=0, n) (b * q^i)` → `b * (1 - q^{n+1}) / (1 - q)`.
* Числа Фибоначчи / Факториалы: Для малых значений компилятор использует Look-up таблицы (готовые массивы), для больших — формулу Бине или итерационный цикл.

4. Геометрические формулы

Они встроены как макросы для быстрой работы с типами num и векторами:

* Расстояние: `dist(p1, p2)` → `\sqrt{(x_2-x_1)^2 + (y_2-y_1)^2}`.
* Площади/Объемы: `S_circle(r) = πr^2`, `V_sphere(r) = \frac{4}{3}πr^3`.
* Треугольники: Теорема косинусов и формула Герона.

5. Аналитический "Движок"

Парсер работает в три шага:

   1. Поиск: Находит структуру в коде (например, `sin(x)/cos(x)`).
   2. Матчинг: Сверяет с таблицей формул.
   3. Замена: Меняет на более эффективный вариант (`tan(x)`).

### Геометрические формулы и функции

Для mclang геометрия — это не просто набор формул, а интеграция алгебраических функций с нашими типами векторов и матриц. Мы сделаем так, чтобы геометрические задачи решались в «тетрадном» стиле, но компилировались в эффективный Си-код.

1. Планиметрия (`std.geom.2d`)

Здесь мы используем наши кортежи и типы num.

| Фигура | Формула в mclang | Описание |
|---|---|---|
| Круг | `S_circle(r) = πr^2` | Площадь круга |
| Треугольник | `S_tri(a, b, c) = \sqrt{p(p-a)(p-b)(p-c)}` | Формула Герона (где `p` — полупериметр) |
| Расстояние | `dist(A, B) = \sqrt{\sum (A_i - B_i)^2}` | Евклидово расстояние (корень из суммы квадратов разностей) |
| Угол | `angle(A, B) = \arccos{\frac{A \cdot B}{|A| \cdot |B|}}` | Угол между двумя векторами в радианах |

2. Стереометрия (`std.geom.3d`)

Благодаря поддержке LaTeX-дробей и степеней, формулы выглядят чисто:

* Сфера: `V = \frac{4}{3}πr^3` и `S = 4πr^2`.
* Конус: `V = \frac{1}{3}πr^2h`.
* Расстояние от точки до плоскости: `d = \frac{|ax_0 + by_0 + cz_0 + d|}{\sqrt{a^2 + b^2 + c^2}}`.

3. Геометрические функции и преобразования

Поскольку мы заложили работу с матрицами, геометрические трансформации становятся нативными:

* Поворот 2D: `rotate(v, α) = R_α * v`, где `$R_\alpha$` — матрица поворота.
* Нормализация: `normalize(v) = \frac{v}{|v|}`.
* Отражение: `reflect(v, n) = v - 2(v \cdot n)n` (стандарт для оптики и физики столкновений).

4. Аналитическая геометрия

Компилятор mclang может упрощать геометрические выражения. Если он видит квадрат расстояния `dist(A, B)^2`, он уберет операцию `sqrt`, оставив только сумму квадратов разностей, что в Си будет работать в разы быстрее.

Итог: как это работает в коде

```
import std.geom.2d as geom

// Точки как векторы
A = [0, 0]B = [3, 4]

// Прямой вызов функции
d = geom.dist(A, B) // В Си станет: sqrt(pow(3-0, 2) + pow(4-0, 2))
```

### Физические формулы

Для mclang встроенные физические формулы — это не просто текст, а оптимизированные пресеты. Пользователь может вызывать их как встроенные макросы или функции, а компилятор развернет их в максимально быстрый Си-код.

Мы разделим их на блоки по разделам физики, используя нашу систему std-импортов и автоматический вывод типов.

1. Механика (`std.physics.mechanics`)

Это база, где активно используются наши векторы и квадраты.

| Физический смысл | Формула в mclang | Трансляция в Си |
|---|---|---|
| Кинетическая энергия | `E_k(m, v) = \frac{mv^2}{2}` | `(m * pow(v, 2)) / 2.0` |
| Сила гравитации | `F_g(m1, m2, r) = G * \frac{m1 * m2}{r^2}` | `G * (m1 * m2 / pow(r, 2))` |
| Путь при ускорении | `s(v0, a, t) = v0 * t + \frac{at^2}{2}` | `v0 * t + (a * pow(t, 2)) / 2.0` |

2. Термодинамика (std.physics.thermo)

Здесь работают наши константы Планка и Больцмана.

* Уравнение Менделеева-Клапейрона:
`P(n, T, V) = \frac{nRT}{V}` (где `R` берется из встроенных таблиц данных).
* Средняя кинетическая энергия молекул:
`E_{avg}(T) = \frac{3}{2} k_B T`.

3. Электродинамика (std.physics.electro)

Тут мы задействуем комплексные числа (если они включены) для расчетов цепей переменного тока.

* Закон Ома: `I = \frac{U}{R}`.
* Закон Кулона: `F_c(q1, q2, r) = k * \frac{|q1 * q2|}{r^2}` (использует наш встроенный `abs` -> `fabs`).
* Энергия конденсатора: `W = \frac{CU^2}{2}`.

4. Квантовая и Релятивистская физика

Благодаря поддержке Unicode и степеней, формулы выглядят как в учебнике:

* Энергия покоя: `E = mc^2`.
* Энергия фотона: `E = h\nu` (используем греческую `ν`).
* Длина волны де Бройля: `\lambda = \frac{h}{p}`.

Как это работает «под капотом» mclang:

   1. Константная база: При использовании `G` или `h` компилятор не ищет их в рантайме, а подставляет значения из встроенной таблицы: `G = 6.67430e-11`.
   2. Проверка размерностей: Если мы ввели поддержку `units`, компилятор не даст тебе подставить секунды вместо массы в формулу `E=mc^2`.
   3. Авто-векторизация: Если ты передаешь в формулу `v` (скорость) как массив `num[]`, компилятор сам решит:
        * Либо вернуть массив энергий для каждой скорости.
        * Либо (если это вектор) посчитать квадрат модуля вектора: `$v^2 = v_x^2 + v_y^2 + v_z^2$`.

Пример использования:

```
import std.physics.mechanics as mech

// Считаем энергию для списка скоростей
mass = 1.5speeds = [10, 20, 30]
energies = mech.E_k(mass, speeds) 
// Результат: [75, 300, 675]
```

### Вызов формул

Чтобы вызвать формулу mclang для рисования круга (например, в JS на <canvas> или в Си с графической библиотекой), нам нужно использовать уравнение окружности или параметрические координаты.

Так как наш язык компилируется в Си, он подготовит быструю функцию, которую основной рантайм будет вызывать в цикле.

1. Математическая модель (mclang)

Создадим файл geometry.mc. Для рисования нам удобнее всего получать координаты `x` и `y` через угол `t`:

```
// geometry.mc
import std.math

// Параметрические координаты точки на окружности
circle_x(r, t, center_x = 0) = center_x + r * cos(t)
circle_y(r, t, center_y = 0) = center_y + r * sin(t)

// Или проверка: находится ли точка внутри круга (для попиксельной отрисовки)
is_inside(x, y, r, cx, cy) = 
    (x - cx)^2 + (y - cy)^2 <= r^2
```

2. Как это будет работать в JS (Canvas)

Компилятор превратит это в WebAssembly. В JavaScript это будет выглядеть так:

```
import { circle_x, circle_y } from './geometry.wasm';
const ctx = canvas.getContext('2d');const R = 100, CX = 250, CY = 250;

ctx.beginPath();// Рисуем круг, вызывая сверхбыструю формулу mclang в циклеfor (let t = 0; t <= 2 * Math.PI; t += 0.01) {
    const x = circle_x(R, t, CX);
    const y = circle_y(R, t, CY);
    ctx.lineTo(x, y);
}
ctx.stroke();
```

3. Как это будет работать в Си (Raylib/SDL)

```
#include "geometry.h"
for (float t = 0; t < 2 * PI; t += 0.01) {
    // Вызов скомпилированного Си-кода из mclang
    float x = circle_x(100, t, 250);
    float y = circle_y(100, t, 250);
    DrawPixel(x, y, RED);
}
```

Почему это круто?

   1. Скорость: Все расчеты sin/cos или x^2 + y^2 происходят в нативном коде, а не в медленном скрипте.
   2. Декларативность: Ты просто описал математику круга, а mclang сам прокинул её во все языки.
   3. Гибкость: Если ты захочешь нарисовать не круг, а сложную эпициклоиду, ты просто поменяешь формулу в .mc файле, не трогая код отрисовки.

Для создания физического движка mclang подходит идеально, потому что он позволяет вынести тяжелые расчеты из медленного JS/Python в быстрый Си, сохраняя при этом «читаемость» формул.

Вот пошаговая стратегия использования mclang как ядра физики:

1. Написание формул (Уровень Math)

Вместо того чтобы писать мешанину из кода, ты описываешь физические законы в .mc файлах. Например, интегратор Верле (стандарт для игр):

```
// physics_core.mcg = 9.806

// Вычисление новой позиции (Verlet Integration)
export get_pos(pos, prev_pos, acc, dt) =
    pos + (pos - prev_pos) + acc * dt^2

// Реакция на столкновение (отскок)
export reflect_vel(v, normal, friction = 0.9) =
    (v - 2 * (v ⋅ normal) * normal) * friction
```

2. Компиляция в Си-библиотеку

Ты компилируешь этот файл. На выходе получаешь .c и .h файлы, где все операции с векторами (v ⋅ normal) уже развернуты в быстрые Си-циклы или SIMD-инструкции.

3. Интеграция в игровой движок## Если пишешь на JS (браузерная игра):

Ты компилируешь ядро в WebAssembly.

* Весь массив частиц (позиции, скорости) хранится в SharedArrayBuffer.
* JS только передает данные в Wasm, где mclang за один проход обсчитывает тысячи столкновений.
* Результат: Ты можешь отрисовывать на Canvas тысячи объектов без лагов.

Если пишешь на Си/C++:

Ты просто подключаешь сгенерированный заголовок.

```
#include "physics_core.h"
void UpdatePhysics(Entity* e, float dt) {
    // Вызов формулы из mclang
    e->new_pos = get_pos(e->pos, e->old_pos, e->acc, dt);
}
```

4. Почему это «киллер-фича» для разработчика:

* Математическая чистота: Твоя физика не замусорена синтаксисом C++ или JS. Видны только формулы.
* Кроссплатформенность: Одна и та же физика работает и в вебе, и на мобилках, и на десктопе.
* Оптимизация: Твой компилятор mclang может автоматически упрощать формулы (например, убирать sqrt там, где можно сравнивать квадраты расстояний), что критично для физики в реальном времени.

### Упрощение вычислений

Упрощение функций (символьные вычисления) — это самая сложная, но и самая впечатляющая часть компилятора. В mclang мы реализуем это на этапе оптимизации AST (Abstract Syntax Tree).

Поскольку мы компилируем в Си, упрощение позволяет нам не просто считать формулу, а генерировать максимально быстрый код, выкидывая лишние операции.

1. Постоянная свертка (Constant Folding)

Это база. Если в формуле есть вычисления, которые можно сделать до запуска программы, компилятор делает их сам.

* mclang: `f(x) = x + (2 * π * 5)`
* Си: `return x + 31.4159265;` (вместо того, чтобы трижды умножать в рантайме).

2. Алгебраические упрощения

Компилятор должен знать законы алгебры, чтобы сокращать выражения:

* Нейтральные элементы:
    * `x + 0` → `x`
    * `x * 1` → `x`
    * `x * 0` → `0`
* Сокращение одинаковых термов:
    * `x + x` → `2x`
    * `x * x` → `x^2`
    * `x - x` → `0`
* Раскрытие скобок и вынос за скобки:
    * `(x + 1) * (x - 1)` → `x^2 - 1`

3. Pattern Matching

Компилятор имеет базу «математических правил». Когда он видит узел дерева, похожий на шаблон, он заменяет его на более простую или удобную для Си форму.

Шаблон: `A^2 - B^2`
Замена: `(A + B) * (A - B)`
Зачем это в Си: Если дальше по коду идет деление на `(x - y)` или `(x + y)`, компилятор увидит это и сократит дробь еще до генерации Си-кода.

Обратная свертка

Шаблон: `(A + B) * (A - B)`
Замена: `A^2 - B^2`
Зачем это в Си: Если дальше по коду нет деления на `(x - y)` или `(x + y)`, компилятор сокращает выражение до простого вычисления `x * x - y * y` на Си-коде.

4. Работа с LaTeX-структурами

Многие функции из тех, что мы обсуждали, могут быть упрощены аналитически.

Перед генерацией кода mclang прогоняет формулу через фильтры.

| Исходный код | Оптимизированный Си-код | Почему |
|---|---|---|
| `(x + 1)^2` | `x*x + 2*x + 1` | Быстрее считать в Си без pow |
| `log(e^x)` | `x` | Аналитическое сокращение |
| `\sqrt{x^2}` | `|x|` | Автоматически вставит `fabs(x)` |
| `e^{ln(x)}` | `x` | Аналитическое сокращение |
| `ln(e^x)` | `x` | Аналитическое сокращение |
| `sin(x)^2 + cos(x)^2` | `1.0` | Тригонометрическое тождество |
| `x/x` | `1.0` | Сокращение (с проверкой на `x != 0`) |

5. Дифференцирование (Symbolic Differentiation)

Если пользователь пишет `\frac{d}{dx} (x^3)`, компилятору mclang нет смысла считать это численно. Он может аналитически превратить это в `3x^2` еще до генерации Си-кода.

6. Зачем это нужно компилятору

Производительность: `x * x` работает быстрее, чем вызов тяжелой функции `pow(x, 2)`.

Стабильность: Упрощение формул позволяет избежать деления на ноль там, где выражение можно сократить аналитически (как в примере `(x^2-1)/(x-1) -> x+1`).

Связь с `from`: Когда ты пишешь `a from y = a^2`, движку упрощения проще «развернуть» формулу, если она приведена к стандартному виду.

В коде компилятора это будет выглядеть как рекурсивная функция.

7. Как это реализовано в архитектуре (Pipeline):

   1. Parser: Строит дерево (AST). Например: `BinaryOp('+', ID('x'), BinaryOp('*', CONST(2), CONST(3)))`.
   2. Optimizer (Simplify): Рекурсивно проходит по дереву. Видит `2 * 3`, заменяет на `6`. Видит `x + 0`, заменяет на `x`.
   3. Generator: Выдает Си-код уже упрощенного дерева.

Пример того, как "умнеет" код:

Вход (mclang):

```
f(x) = \frac{x^2 - 1}{x - 1} + 0 * sin(x)
```

Без упрощения (Си):

```
return (pow(x, 2) - 1) / (x - 1) + 0 * sin(x); // Медленно, риск деления на 0
```

С упрощением mclang (Си):

```
return x + 1; // Молниеносно и безопасно
```

### Решение уравнений и неравенств

Для встраиваемого языка, компилируемого в Си, решение уравнений и неравенств может быть реализовано двумя способами: статическим (на этапе компиляции) и динамическим (через численные методы в рантайме).

Поскольку mclang — это «чистый вычислитель», мы внедрим оба механизма под общим декларативным синтаксисом.

1. Синтаксис «Найти» (`solve`)

Для поиска корней уравнения `f(x) = 0` или решения системы уравнений мы будем использовать конструкцию `solve`.

```
// Найти x, при котором формула равна нулю
x_root = solve(x) { x^2 - 4 == 0 } 

// С указанием диапазона (для численных методов)
x_opt = solve(x in [0, 10]) { sin(x) == 0.5 }
```

2. Численные методы (Runtime)

Поскольку в Си нет встроенного решателя, компилятор mclang при встрече `solve` делает следующее:

   1. Генерирует функцию из выражения внутри `{}`.
   2. Вставляет вызов встроенной библиотеки численных методов (например, метод Ньютона или метод бисекции).

Пример в Си:

```
// mclang превращает solve в вызов функции библиотеки mclang_runtime
double root = mc_solve_newton(my_formula_ptr, initial_guess);
```

3. Решение неравенств

В контексте библиотеки-вычислителя решение неравенства — это чаще всего поиск интервала или маски (для массивов).

* Для скаляров: `solve` вернет границы интервала (массив из двух чисел).
* Для массивов (Векторный поиск):

```
// Найти все индексы, где выполняется условие
indices = solve(i in data) { data[i] > 0 }
```

4. Аналитическое упрощение (Compile-time)

Если уравнение простое (линейное или квадратное), компилятор mclang не будет подключать тяжелые численные методы. Он сам «выведет» формулу корней.

* Ввод: `solve(x) { ax + b == 0 }`
* Итог в Си: `double x = -b / a;` (с проверкой на `a != 0`).

5. Системы уравнений

mclang сможет решать системы через матричные операции, которые мы уже добавили:

```
// Система: 
// x + y = 10
// x - y = 2
results = solve(x, y)
    x + y == 10
    x - y == 2
```

Компилятор переведет это в матричный вид `Ax = B` и решит через метод Гаусса.

Справочные данные:

   1. Численные методы решения уравнений — методы Ньютона и секущих для реализации в Си.
   2. Решение систем линейных уравнений — база для реализации solve для матриц.

### Квадратные уравнения

Для квадратных уравнений в mclang мы внедрим «умный» аналитический модуль. Компилятор не будет мучить процессор численными методами там, где есть готовая формула.

1. Синтаксис

Пользователь просто описывает уравнение, а компилятор распознает его тип:

```
// mclang сам поймет, что это квадратное уравнение
[x1, x2] = solve(x) { ax^2 + bx + c == 0 }
```

2. Как это работает в компиляторе (Логика)

Парсер анализирует AST (дерево) выражения. Если максимальная степень переменной x равна 2, компилятор не генерирует итерационный цикл, а сразу вставляет в Си-код классический алгоритм:

   1. Вычисление дискриминанта: `D = b^2 - 4ac`.
   2. Проверка `D`:
        * Если `D > 0`: вычисляются два корня.
        * Если `D = 0`: один корень.
        * Если `D < 0`: возвращается nan или комплексные числа (если включена поддержка).
   
3. Генерация Си-кода

На выходе получается максимально быстрый и стабильный код:

```
double D = b * b - 4 * a * c;if (D > 0) {
    double sqrtD = sqrt(D);
    x1 = (-b + sqrtD) / (2 * a);
    x2 = (-b - sqrtD) / (2 * a);
} else if (D == 0) {
    x1 = x2 = -b / (2 * a);
} else {
    x1 = x2 = NAN; // Или работа с комплексными
}
```

4. Автоматическое приведение (Simplification)

Даже если уравнение записано «грязно», mclang сначала упростит его:

* Ввод: `solve(x) { x^2 + 2x == 5x - 2 }`
* Препроцессор: Перенесет всё в левую часть: `x^2 - 3x + 2 == 0`.
* Итог: Сгенерирует код для `a=1, b=-3, c=2`.

5. Комплексные корни

Если мы работаем в режиме «физика/математика», мы можем возвращать структуру complex:

```
[x1, x2] = solve(x: complex) { x^2 + 1 == 0 }
// x1 = 0 + 1i, x2 = 0 - 1i
```

### Выражение переменных через равенства

Это напоминает реактивное связывание или аналитическое вычисление зависимостей. В математике мы часто говорим: «Найдем X, при условии что A выражено через Y».

Чтобы это работало в mclang, препроцессор должен уметь «разматывать» цепочку зависимостей в обратном порядке, чтобы в Си получилась правильная последовательность инструкций.

Как это будет работать (Синтаксис)

Ключевое слово from указывает компилятору, что переменную a нужно вычислить прямо сейчас, используя следующее за ним выражение.

```
func(x, y) =
  x = a^2               // 2. Считаем результат
  a from y = \sqrt{a}   // 1. Выражаем 'a' из этого равенства
```

Что сделает компилятор (Логика)

    1. Поиск зависимости: Видит a from y = ....
    2. Решатель (Solver): Включает встроенный аналитический движок, чтобы выразить `a` через `y`.
    3. Компилятор понимает, что если `y = \sqrt{a}`, то `a = y^2`

В Си-код пойдет:

```
double a = pow(y, 2); // Результат вывода из 'a from y = sqrt(a)'
double x = pow(a, 2);
return x;
```

Правила для парсера:

Синтаксис: `ИмяПеременной from Уравнение`.
Действие: Вызвать модуль `Simplify/Solve` для этого уравнения относительно `ИмяПеременной`.
Ошибка: Если уравнение нельзя решить аналитически, выдать ошибку на этапе компиляции (или предложить численный метод `solve`).

### Встроенные функции

Это сделает mclang не просто языком, а огромным справочником, где формулы не нужно переписывать — их достаточно просто вызвать.
Для этого мы внедрим стандартную библиотеку std, в которой формулы хранятся в их «эталонном» виде, а компилятор сам выводит нужную переменную.

1. Как это работает (Синтаксис)

Вместо того чтобы гуглить формулу, ты используешь пространство имен std и ключевое слово from.

```
// Пример 1: Закон Ома (std.physics.ohm)
// В библиотеке он записан как: I = U / R
get_voltage(current, resistance) = 
    u from std.physics.ohm(I=current, R=resistance)

// Пример 2: Геометрия (std.geom.circle)
// В библиотеке: S = π * r^2
get_radius(area) = 
    r from std.geom.circle(S=area)
```

2. Магия компилятора (Analytic Derivation)

Когда ты пишешь `r from std.geom.circle(S=area)`, компилятор делает следующее:

   1. Находит эталонную формулу: `S = \pi \cdot r^2`.
   2. Смотрит, что дано: `S = area`.
   3. Смотрит, что нужно найти: `r`.
   4. Символьно выводит формулу: `r = \sqrt{\frac{S}{\pi}}`.
   5. Генерирует чистый Си-код: `double r = sqrt(area / 3.14159);`.

3. Основные встроенные модули

Мы заложим в базу самые популярные блоки:

* `std.math`:
    * `pythagoras(a, b, c)` → `$a^2 + b^2 = c^2$`
    * `quadratic(a, b, c, x)` → `$ax^2 + bx + c = 0$`
* `std.physics`:
    * `ohm(U, I, R)` → `$I = U / R$`
    * `newton2(F, m, a)` → `$F = m \cdot a$`
    * `einstein(E, m, c)` → `$E = m \cdot c^2$`
* `std.geom`:
    * `circle(S, r, d, L)` → объединяет формулы площади, радиуса, диаметра и длины окружности.
    * `sphere(V, S, r)` → объем и площадь поверхности сферы.

4. Почему это удобно?

Тебе не нужно помнить, надо ли делить или умножать. Ты просто указываешь сущность и известные параметры, а mclang сам строит уравнение.

Пример сложного вызова:

```
// Найти массу объекта, зная его кинетическую энергию и скорость
m_from_energy(E, v) = 
    m from std.physics.mechanics.kinetics(E_k=E, v=v)
```

В Си пойдет: `return (2 * E) / pow(v, 2)`;

Итог:

   1. Библиотека `std`: Набор текстовых файлов с эталонными формулами.
   2. Solver-модуль: Аналитическое ядро, которое умеет выражать одну переменную через другие.
   3. Namespace-парсер: Чтобы понимать записи типа `std.physics.ohm`.

## 9. Архитектура и Runtime

* Pure Logic: Полное отсутствие print, input и других побочных эффектов. Только чистые функции.
* Нулевые зависимости: Сгенерированный Си-код автономен. Он не требует тяжелых внешних библиотек и может работать даже на микроконтроллерах.
* Управление памятью: Отказ от динамической памяти (malloc). Все вычисления происходят на стеке для максимальной скорости и безопасности.

Встраиваемость (Embeddable):

* Export: Все функции верхнего уровня автоматически попадают в `.h` файл.
* No Runtime: Сгенерированный код не требует внешних библиотек (кроме `math.h`).
* JS: Компилируется в WASM за секунды.
* Python: Подключается как нативный модуль через `ctypes` или `C API`.

## 10. Система встраиваемых модулей

### Импорт

Если ты написал крутую формулу для расчета сопротивления воздуха в одном файле, ты захочешь импортировать её в другой проект, а не копипастить код.

Также это нужно для связи с математическими константами или специфическими функциями Си-библиотек (например, если нужна очень специфическая функция из `math.h`). Поскольку мы компилируем всё в Си, система импортов будет работать на двух уровнях: внутреннем (между .mc файлами) и внешнем (связь с нативным Си). Это позволяет разбивать проект на логические модули. Например, в одном файле константы, в другом — физика.

```
// constants.mc
G = 6.674e-11c = 299792458
```

```
// physics.mc
import "./constants.mc"

force(m1, m2, r) = G * \frac{m1 * m2}{r^2}
```

Как это работает:

Препроцессор находит `import`, читает указанный файл и добавляет его функции и константы в текущую область видимости. При генерации Си-кода это превратится в `#include "constants.h"`.

Если файл огромный, а тебе нужна только одна формула:

```
from "./fluid_dynamics.mc" import drag_coefficient
```

Иногда нужно вызвать функцию, которой нет в mclang, но она есть в стандартной библиотеке Си или написана тобой на Си вручную.

```
// Объявляем, что функция существует где-то вовне (в Си)
extern func erf(x: num) : num

// Теперь её можно использовать в mclang
normal_dist(x) = 0.5 * (1 + erf(x / \sqrt{2}))
```

Как это работает:

`extern` говорит компилятору: «Не ищи определение этой функции здесь, просто пропиши её вызов, а линковщик Си разберется». Это открывает доступ ко всем мощностям `math.h`, `GSL` (GNU Scientific Library) или `FFTW`.

Для специфических системных вещей можно разрешить прямые инклюды, которые препроцессор просто перенесет в начало .c файла:

```
import "math.h"  // Станет #include <math.h> в Си
```

Если в двух разных модулях есть функция с именем calculate, можно использовать алиасы:

```
import "./astronomy.mc" as astro
import "./ballistics.mc" as ball

res = astro.calculate(x) + ball.calculate(y)
```

Главный нюанс:

Так как mclang — это «язык-библиотека», при импорте одного .mc файла в другой, компилятор должен следить, чтобы не возникло циклических зависимостей (A импортирует B, а B импортирует A).

Стандартную библиотеку `std`, не нужно импортировать явно. Там будут лежать `π`, `e`, `sin`, `cos` и прочие базовые вещи.

### Экспорт

В нашей текущей декларативной концепции «один файл — одна библиотека функций», экспорт должен работать максимально незаметно для пользователя, но строго для компилятора Си.

Поскольку мы отказались от лишних ключевых слов, предлагаю два уровня экспорта:

1. Автоматический экспорт (Default)

По умолчанию все функции, определенные на верхнем уровне (без отступа), считаются экспортируемыми.

* Если ты написал `f(x) = x^2`, компилятор автоматически добавит её прототип в `.h` файл.
* Это идеально для простых математических модулей.

2. Приватные функции (Локальные)

Иногда нужны вспомогательные функции, которые не должны быть видны в JS или Python. Для этого можно использовать нижнее подчеркивание:

* `_internal_calc(a) = a / 2` — компилируется в Си как static, не попадает в заголовочный файл и недоступна снаружи.

Как это выглядит в итоговом продукте

Компилятор mclang при обработке файла `orbits.mc` генерирует два файла:

orbits.h (Интерфейс для C/JS/Python):

```
// Здесь только то, что мы экспортируем
double calculate_velocity(double mass, double radius);
```

orbits.c (Реализация):

```
#include "orbits.h"
#include <math.h>
// Приватная функция (static), ее нет в .h
static double _get_constant() {
    return 6.674e-11;
}
double calculate_velocity(double mass, double radius) {
    return sqrt(_get_constant() * mass / radius);
}
```

### Итог по экспорту:

* Публичные: Все функции с обычными именами.
* Приватные: Имена, начинающиеся с `_`.
* Результат: Чистый заголовочный файл Си, готовый к линковке.

## 10. Примеры

### Пример трансформации

Исходный код на mclang (.mc):

```
calculate_orbit(α, x)
    if (x > 0)
        2π * x^2 + \frac{α, 2}x
    else
        0
```

Результат для Си (.c):

```
#include <math.h>
double calculate_orbit(double alpha, double x) {
    if (x > 0) {
        return (
            2 * 3.14159265 * pow(x, 2) + (alpha / 2.0) * x
        );
    } else {
        return 0;
    }
}
```

### Пример финального кода (.mc)

Файл `ballistics.mc`:

```
// Константы
g = 9.806
π = 3.14159

// Определение функции
trajectory(v0, angle, t) = 
    if t < 0
        0
    else
        v0 * t * sin(rad) - \frac{gt^2, 2}
    where
        rad = angle * \frac{π, 180}
```

Трансляция в Си (автоматически при компиляции):

```c
// mclang сам добавит нужные инклуды и обертки
double trajectory(double v0, double angle, double t) {
    if (t < 0) {
        return 0;
    } else {
        double rad = angle * (3.14159 / 180.0);
        return v0 * t * sin(rad) - (9.806 * pow(t, 2) / 2.0);
    }
}
```
