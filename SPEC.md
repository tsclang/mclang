# MClang Language Specification v0.1

## 1. Обзор

**mclang** (Math C Language) — специализированный язык для описания математических вычислений, транслируемый в чистый код на языке Си без внешних зависимостей.

Файлы исходного кода имеют расширение **`.mc`** (пример: `physics.mc`, `constants.mc`).

### 1.1 Назначение

Язык-вычислитель, а не язык общего назначения. Не предназначен для I/O, UI или сетевого взаимодействия. Цель — принять числовые аргументы, вычислить формулу и вернуть результат.

### 1.2 Целевые платформы

| Таргет | Команда | Результат |
|--------|---------|-----------|
| PC / JS / Python | `mclang file.mc --target wasm` | `.wasm` |
| Python (C extension) | `mclang file.mc --target shared` | `.so` / `.dll` + `_loader.py` |
| C / C++ | `mclang file.mc --target c` | `.c` + `.h` |
| Node.js (N-API) | `mclang file.mc --target node` | `.c` + `.h` + `_napi.c` + `binding.gyp` + `_bindings.js` |
| Rust FFI | `mclang file.mc --target rust` | `.c` + `.h` + `_bindings.rs` |
| STM32 / ESP32 | `mclang file.mc --target avr --precision f32` | `.c` + `.h` |
| Arduino / 8-bit | `mclang file.mc --target avr --precision fixed` | `.c` + `.h` |

### 1.3 Модель интеграции

Один `.mc`-файл компилируется в одну библиотеку. Таргет выбирается при компиляции — исходник не меняется.

#### JavaScript / WebAssembly

```
mclang physics.mc --target wasm
```

```javascript
import { trajectory, impact_force } from './physics.wasm';

const height = trajectory(1.5, 20.0, 45.0);
console.log(`Высота: ${height}м`);
```

#### Python (C extension)

```
mclang physics.mc --target shared
```

```python
import ctypes
mc = ctypes.CDLL('./physics.so')
mc.impact_force.restype = ctypes.c_double
mc.impact_force.argtypes = [ctypes.c_double, ctypes.c_double, ctypes.c_double]

force = mc.impact_force(70.5, 15.0, 0.1)
print(f"Сила: {force} Н")
```

Или через cffi / nanobind (обёртка генерируется компилятором при `--target shared`):

```python
import mclang_physics as mc

force = mc.impact_force(mass=70.5, velocity=15.0, time=0.1)
print(f"Сила: {force} Н")
```

#### C / C++

```
mclang physics.mc --target c
```

```c
#include "physics.h"

int main(void) {
    double force = impact_force(70.5, 15.0, 0.1);
    printf("Сила: %f Н\n", force);
    return 0;
}
```

Компиляция:
```
gcc main.c physics.c -o app -lm
```

## 2. Типы данных

### 2.1 Примитивные типы

| Тип mclang | Описание | Трансляция (PC/JS) | Трансляция (AVR f32) | Трансляция (8-bit) |
|-----------|---------|-------------------|---------------------|-------------------|
| `num` | Число (адаптивная точность) | `double` | `float` | `int16_t` (fixed-point) |
| `int` | Целое (только для индексов/счётчиков) | `int` | `int` | `int` |

Пользователь не указывает типы явно для математических переменных. Тип `int` — только для счётчиков и индексов.

### 2.2 Адаптивный тип `mc_num`

В сгенерированном `.h`-файле:

```c
#ifdef MC_USE_FAST_FLOAT
    typedef float mc_num;
#elif defined(MC_USE_8BIT)
    typedef int16_t mc_num;
#else
    typedef double mc_num;
#endif
```

### 2.3 Составные типы

- `num[]` — одномерный массив (вектор)
- `num[][]` — двумерный массив (матрица)
- Статический размер `num[3]` → `double v[3]` на стеке; параметр `_len` **не** генерируется
- Статический двумерный размер `num[3][4]` → `mc_num v[3*4]`; параметры `_rows`/`_cols` **не** генерируются, `m[i][j]` → `m[i*4+j]` (ширина известна)
- Динамический размер → указатель + неявный параметр длины (только если необходимо)

### 2.4 Передача массивов в функции

Для каждого параметра-массива компилятор автоматически генерирует скрытый параметр длины `имя_len: int`. Пользователь пишет `.length` — компилятор подставляет нужный `_len`.

```
// mclang
total_energy(masses: num[], velocities: num[]) =
    ∑(i=0, masses.length - 1) \frac{masses[i] * velocities[i]^2}{2}
```

```c
// Си (сгенерированный код)
mc_num total_energy(mc_num* masses, int masses_len,
                    mc_num* velocities, int velocities_len) {
    mc_num _acc = 0;
    for (int i = 0; i <= masses_len - 1; i++)
        _acc += (masses[i] * pow(velocities[i], 2)) / 2.0;
    return _acc;
}
```

Правила:
- Статический размер `num[3]` → параметр длины не генерируется (длина известна на этапе компиляции)
- `.length` всегда ссылается на скрытый `_len` соответствующего массива
- Если два массива должны иметь одинаковую длину — компилятор добавляет runtime-guard: `if (masses_len != velocities_len) return NAN;`

### 2.5 Передача матриц в функции

Для `num[][]` компилятор генерирует два неявных параметра: `имя_rows` и `имя_cols`.

```
f(m: num[][]) = ...
```

```c
mc_num f(mc_num* m, int m_rows, int m_cols)
```

- `.rows` → `m_rows`, `.cols` → `m_cols`
- `.length` для матрицы недопустим — **ошибка компиляции**
- Элемент `m[i][j]` → `m[i * m_cols + j]` (плоский массив)

### 2.6 Множественный возврат

Функция может вернуть несколько значений через массив-литерал:

```
roots(a, b, c) = \pm \sqrt{b^2 - 4*a*c} - b / (2*a)
    where
        b^2 - 4*a*c >= 0
```

`\pm expr` — синтаксический сахар, разворачивается в `[(+expr), (-expr)]`.

Общий случай — явный возврат массива:

```
min_max(v: num[]) = [min_val, max_val]
    where
        min_val = ...
        max_val = ...
```

В Си функция возвращает указатель на статический массив:

```c
mc_num* roots(mc_num a, mc_num b, mc_num c) {
    static mc_num _result[2];
    mc_num _d = sqrt(pow(b, 2) - 4*a*c);
    _result[0] = (+_d - b) / (2*a);
    _result[1] = (-_d - b) / (2*a);
    return _result;
}
```

Вызывающий код получает `num[]` и обращается по индексу: `r[0]`, `r[1]`.

**Ограничение:** размер возвращаемого массива должен быть известен на этапе компиляции (статический).

## 3. Синтаксис

### 3.1 Структура файла

```
[глобальные константы]
[импорты]
[объявления функций]
```

### 3.2 Объявление функций

```
имя(параметры) = выражение
```

Многострочная форма (блок через отступы):

```
имя(параметры) =
    выражение1
    выражение2
    результат
```

Возврат значения — **неявный**: последнее вычисленное выражение является результатом. Ключевое слово `return` отсутствует.

Рекурсия поддерживается. Компилятор генерирует форвард-декларации всех функций, что позволяет прямую и взаимную рекурсию. Хвостовая рекурсия не оптимизируется самим компилятором mclang, но C-компилятор делает это при флаге `-O2`.

### 3.3 Мутабельность переменных

| Переменная | Мутабельность | Нарушение |
|-----------|--------------|-----------|
| Параметр функции | Иммутабельна | Ошибка компиляции |
| Локальная переменная | Мутабельна | — |
| Глобальная константа | Иммутабельна | Ошибка компиляции |

Правила:
- Локальная переменная **должна быть инициализирована** перед первым использованием — иначе ошибка компиляции
- Переприсваивание локальной переменной разрешено

```
sum_sq(n: int) =
    res = 0              // инициализация — ok
    for i in 1..n
        res = res + i^2  // переприсваивание — ok
    res

bad(x) =
    x = x + 1   // ОШИБКА: параметр иммутабелен
    x
```

### 3.4 Значимые отступы

Блоки определяются отступами (Python-style). Препроцессор генерирует `{}` для Си.

### 3.5 Автоматическая расстановка точки с запятой (ASI)

Точка с запятой **ставится** в конце строки, если строка содержит законченное выражение.

Точка с запятой **не ставится**, если:
- строка заканчивается на: `+ - * / , = ( [ {` или ключевые слова `if else for while`
- следующая строка начинается на: `+ - * / . ->`
- строка пустая, является комментарием или директивой `#include`

### 3.6 Перенос строк

Если строка заканчивается на оператор или содержит незакрытую скобку — препроцессор считает, что выражение продолжается на следующей строке.

### 3.7 Комментарии

```
// Однострочный комментарий
# Альтернативный однострочный комментарий (Python-style)
```

## 4. Идентификаторы и Unicode

### 4.1 Разрешённые символы

Идентификаторы могут содержать буквы Unicode, в том числе греческие. Препроцессор транслитерирует их в ASCII для Си:

- `π` → `__uni_pi` (или `_u03C0`)
- `α` → `__uni_alpha`

### 4.2 Нижние индексы

| Запись в mclang | Трансляция |
|----------------|-----------|
| `x_{i}` (массив) | `x[i]` |
| `x_{start}` (имя) | `x_start` |
| `H_2O` | `H_2O` (имя переменной) |

### 4.3 Встроенные символьные константы

| Символ | Значение |
|--------|---------|
| `π` / `pi` | 3.14159265358979... |
| `e` | 2.71828182845904... |
| `τ` / `tau` | 6.28318530717958... (= 2π) |
| `φ` / `phi` | 1.61803398874989... (золотое сечение) |
| `inf` / `∞` | `INFINITY` |
| `nan` | `NAN` |

## 5. Константы

### 5.1 Иерархия приоритетов

1. Локальная переменная (`where`-блок или тело функции)
2. Пользовательская глобальная константа (верхний уровень файла)
3. Встроенная константа

### 5.2 Глобальные константы

Присваивание на верхнем уровне файла создаёт неизменяемую константу:

```
G = 6.67430e-11
c = 299792458
```

В Си: `const mc_num G = 6.67430e-11;` или `#define G 6.67430e-11` (для 8-bit).

Переопределение встроенной константы допускается только на уровне модуля:
```
π = 3.14  // Переопределение для всего файла
```

Попытка переопределить константу внутри функции — **ошибка компиляции** (не в `where`-блоке).

### 5.3 Локальное переопределение через `where`

```
my_func(x) = x * π
    where
        π = 10  // Warning: перекрытие стандартной константы
```

### 5.4 Constant Folding

Компилятор вычисляет выражения из одних констант на этапе сборки:
- `2π * r` → `6.28318... * r` (без умножения в рантайме)
- Константа, используемая один раз → подставляется числом
- Константа, используемая многократно → выносится в секцию read-only памяти

## 6. Математический синтаксис

### 6.1 Неявное умножение

Парсер вставляет `*` когда после разобранного выражения следует один из токенов:

| Следующий токен | Пример | Результат |
|-----------------|--------|-----------|
| Идентификатор | `2x`, `π r`, `a b` | `2*x`, `π*r`, `a*b` |
| `(` | `2(a+b)`, `f(x)(y)` | `2*(a+b)`, `f(x)*y` |
| `\sqrt{...}` | `2\sqrt{x}` | `2*sqrt(x)` |
| `\frac{...}{...}` | `2\frac{a}{b}` | `2*(a/b)` |
| `\sin`, `\cos`, `\tan`, … | `2\sin{x}`, `r\cos{θ}` | `2*sin(x)`, `r*cos(θ)` |
| `\lfloor`, `\lceil` | `2⌊x⌋` | `2*floor(x)` |

Приоритет неявного умножения равен приоритету явного `*` и `/`.

**Числа не создают неявное умножение с другими числами:** `2 3` — ошибка синтаксиса.

#### Греческие буквы и ASCII-идентификаторы

Лексер формирует идентификаторы жадно: греческая буква и следующая за ней ASCII-буква без пробела сливаются в **один** токен.

| Запись | Токены | Результат |
|--------|--------|-----------|
| `2πd` | `2`, `πd` | `2 * πd` — `πd` одна переменная ⚠ |
| `2π d` | `2`, `π`, `d` | `2 * π * d` ✓ |
| `2π*d` | `2`, `π`, `*`, `d` | `2 * π * d` ✓ |

Правило: после греческой буквы отделяй латинские символы пробелом или явным `*`.

#### Примеры с несколькими множителями

```
2π d             →  2 * π * d
2π d \cos{θ}     →  2 * π * d * cos(θ)
\frac{2π d}{λ}   →  (2 * π * d) / λ
r^2 \sin{θ}      →  pow(r, 2) * sin(θ)
```

### 6.2 Дроби

```
\frac{числитель}{знаменатель}
```

Трансляция: `((числитель) / (знаменатель))`

Пример: `\frac{x + 1}{y - 1}` → `((x + 1) / (y - 1))`

### 6.3 Степени

| Синтаксис mclang | Трансляция в Си |
|-----------------|----------------|
| `x^2` | `pow(x, 2)` |
| `x^{a + b}` | `pow(x, (a + b))` |
| `x**n` | `pow(x, n)` |

Приоритет степени — наивысший. `2x^2` → `2 * pow(x, 2)`.

### 6.4 Корни

| Синтаксис mclang | Трансляция в Си |
|-----------------|----------------|
| `\sqrt{x}` | `sqrt(x)` |
| `\sqrt[3]{x}` | `pow(x, 1.0/3.0)` |
| `\sqrt[n]{x + 1}` | `pow(x + 1, 1.0/n)` |

### 6.5 Тригонометрия

Функции принимаются как с префиксом `\`, так и без него. Полная таблица синонимов:

| Функция | mclang | Трансляция в Си |
|---------|--------|----------------|
| Тангенс | `tan`, `tg`, `\tan`, `\tg` | `tan(x)` |
| Котангенс | `cot`, `ctg`, `\cot`, `\ctg` | `(1.0 / tan(x))` |
| Арксинус | `arcsin`, `asin`, `\arcsin` | `asin(x)` |
| Арккосинус | `arccos`, `acos`, `\arccos` | `acos(x)` |
| Арктангенс | `arctan`, `arctg`, `atan`, `\arctan`, `\arctg` | `atan(x)` |
| Арктангенс двух аргументов | `arctan2(y, x)`, `atan2(y, x)` | `atan2(y, x)` |
| Арккотангенс | `arccot`, `arcctg`, `\arccot`, `\arcctg` | `atan2(1.0, x)` |
| Гиперб. синус | `sinh`, `sh`, `\sinh`, `\sh` | `sinh(x)` |
| Гиперб. косинус | `cosh`, `ch`, `\cosh`, `\ch` | `cosh(x)` |
| Гиперб. тангенс | `tanh`, `th`, `\tanh`, `\th` | `tanh(x)` |
| Гиперб. котангенс | `coth`, `cth`, `\coth`, `\cth` | `(1.0 / tanh(x))` |
| Секанс | `sec`, `\sec` | `(1.0 / cos(x))` |
| Косеканс | `csc`, `\csc` | `(1.0 / sin(x))` |
| Ареасинус гиперб. | `asinh`, `arcsh`, `\arsinh` | `asinh(x)` |

| Ареатангенс гиперб. | `atanh`, `arcth`, `\artanh` | `atanh(x)` |

### 6.6 Логарифмы

| Синтаксис mclang | Трансляция в Си |
|-----------------|----------------|
| `ln(x)` / `log(x)` | `log(x)` |
| `log10(x)` / `\lg{x}` | `log10(x)` |
| `\log_{base}{x}` / `\log{base}{x}` | `(log(x) / log(base))` |

### 6.7 Прочие функции

| Синтаксис mclang | Трансляция в Си |
|-----------------|----------------|
| `exp(x)` / `e^x` | `exp(x)` |
| `abs(x)` / `\|x\|` / `\abs{x}` | `fabs(x)` |
| `floor(x)` | `floor(x)` |
| `ceil(x)` | `ceil(x)` |
| `round(x)` | `round(x)` |
| `trunc(x)` | `trunc(x)` | усечение к нулю (C99) |
| `cbrt(x)` | `cbrt(x)` | кубический корень (C99) |
| `log2(x)` | `log2(x)` | логарифм по основанию 2 (C99) |
| `hypot(x, y)` | `hypot(x, y)` | гипотенуза √(x²+y²) (C99) |
| `erfc(x)` | `erfc(x)` | дополнение к функции ошибок (C99) |
| `deg(x)` / `\deg{x}` | `x * (π/180)` | градусы → радианы |

### 6.8 Модуль (детали парсинга)

Символ `|` неоднозначен: может быть границей модуля или частью `||` (логическое ИЛИ). Лексер разрешает неоднозначность по контексту предыдущего токена — без стека состояний, O(1).

**Контекст предыдущего токена:**

| Контекст | Предыдущий токен |
|----------|-----------------|
| OPERATOR | оператор (`+ - * / ^ % , = < > ! ( [ {`), ключевое слово, начало строки |
| VALUE | число, идентификатор, `)`, `]` |

**Правила:**

| Ситуация | Трактовка |
|----------|-----------|
| `|` в OPERATOR-контексте | открывающий модуль |
| `|` в VALUE-контексте | закрывающий модуль |
| `||` где первый `|` в VALUE-контексте | логическое ИЛИ |

**Примеры:**

```
|x|            →  abs(x)
x || y         →  x OR y
|x| || |y|     →  abs(x) OR abs(y)
||x| - |y||    →  abs(abs(x) - abs(y))
f(|x|)         →  f(abs(x))
a + |b|        →  a + abs(b)
```

## 7. Операторы

### 7.1 Арифметические

`+`, `-`, `*`, `/`, `%` (остаток), `^` / `**` (степень)

Синонимы:
- `⋅` → `*` для скаляров; dot product / матричное умножение для массивов (dispatch по типу)
- `÷` → `/` для скаляров
- `a \mod b` → `fmod(a, b)`

Остаток от деления:
- оба операнда `int`: `%`
- хотя бы один `num`: `fmod(x, y)`
- Альтернатива: `mod(x, y)`

Дополнительные операторы:

| Оператор | Трансляция в Си | Примечание |
|---------|----------------|-----------|
| `n!` | цикл или lookup-таблица | постфиксный, только `int` |
| `\binom{n}{k}` | `factorial(n)/(factorial(k)*factorial(n-k))` | |
| `\min(a, b)`, `min(a, b)` | `fmin(a, b)` | |
| `\max(a, b)`, `max(a, b)` | `fmax(a, b)` | |
| `sgn(x)`, `sign(x)` | `(x > 0) - (x < 0)` | возвращает `-1`, `0`, `1` |
| `\lfloor x \rfloor` | `floor(x)` | LaTeX-скобки |
| `\lceil x \rceil` | `ceil(x)` | LaTeX-скобки |
| `\lVert v \rVert` | `mc_norm(v, v_len)` | норма вектора |
| `30°` | `30 * (π / 180)` | символ `°` U+00B0 |
| `\gcd(a, b)`, `gcd(a, b)` | алгоритм Евклида | только `int` |
| `\lcm(a, b)`, `lcm(a, b)` | `a * b / gcd(a, b)` | только `int` |
| `\erf{x}`, `erf(x)` | `erf(x)` | math.h C99 |
| `\Gamma{x}`, `gamma(x)` | `tgamma(x)` | math.h C99 |
| `\bar{v}`, `mean(v)` | среднее `num[]` | генерирует цикл |
| `\sigma{v}`, `std(v)` | стандартное отклонение `num[]` | генерирует цикл |
| `\pm expr` | `[+expr, -expr]` | возвращает `num[2]` |
| `\mp expr` | `[-expr, +expr]` | возвращает `num[2]`, обратный порядок |
| `\inf(v)` | `min(v)` | инфимум (≡ `min` для конечных множеств) |
| `\sup(v)` | `max(v)` | супремум (≡ `max` для конечных множеств) |

### 7.2 Сравнение

| Оператор mclang | Оператор Си | Результат |
|----------------|------------|---------|
| `==` | `==` | `1.0` или `0.0` |
| `!=`, `<>`, `\neq`, `\ne` | `!=` | `1.0` или `0.0` |
| `>`, `<` | `>`, `<` | `1.0` или `0.0` |
| `>=`, `≥`, `\geq` | `>=` | `1.0` или `0.0` |
| `<=`, `≤`, `\leq` | `<=` | `1.0` или `0.0` |

**Цепочки сравнений:** `0 < x < 10` → `(0 < x && x < 10)`

### 7.3 Логические

| Оператор mclang | Си-эквивалент | Примечание |
|----------------|--------------|-----------|
| `and`, `&&`, `∧` | `&&` | Ленивые вычисления |
| `or`, `\|\|`, `∨` | `\|\|` | Ленивые вычисления |
| `xor`, `⊕` | `((a!=0)^(b!=0))?1.0:0.0` | Логический XOR |
| `not`, `!`, `¬` | `!` | Высокий приоритет |

Битовых операций нет.

### 7.4 Принадлежность (`∈`)

| Синтаксис | Трансляция в Си |
|-----------|----------------|
| `x ∈ [a, b]` / `x in [a, b]` | `(x >= a && x <= b)` |
| `x ∈ (a, b)` | `(x > a && x < b)` |
| `x ∈ ℕ` / `x ∈ \mathbb{N}` | `(x >= 0 && fmod(x, 1) == 0)` |
| `x ∈ ℤ` / `x ∈ \mathbb{Z}` | `(fmod(x, 1) == 0)` |
| `x ∈ ℝ` / `x ∈ \mathbb{R}` | `isfinite(x)` (NaN и Inf не входят в ℝ) |
| `x ∈ ℂ` / `x ∈ \mathbb{C}` | всегда истина (mclang не имеет комплексного типа; guard не эмитируется) |
| `x ∉ ...` / `x !in ...` | отрицание вышеперечисленного |

## 8. Управляющие конструкции

### 8.1 `if / else`

Единственный канонический синтаксис — **`if-first`**. Форма `value if cond` не поддерживается.

Инлайн-форма (однострочный блок):
```
check(x) = if (x < 0) -1 else sqrt(x)
```
→ тернарный оператор Си: `(x < 0) ? -1 : sqrt(x)`

Блочная форма:
```
step_function(x) =
    if x < 0
        -1
    else if 0 ≤ x < 10
        x^2
    else
        100
```

Guard clause (без else):
```
sgn(x) =
    if (x > 0) 1
    if (x < 0) -1
    0
```
→
```c
double sgn(double x) {
    if (x > 0) return 1;
    if (x < 0) return -1;
    return 0;
}
```

**Правило препроцессора:** если после `if (условие)` на той же строке есть значение — это однострочный блок с `return`.

### 8.1.1 Кусочные функции (`\begin{cases}`)

LaTeX-альтернатива `if/else`-цепочке. Каждый случай: `выражение & условие \\`. Последний случай без условия — ветка `else`.

```
f(x) = \begin{cases}
    x^2      & x < 0 \\
    0        & x == 0 \\
    \sqrt{x}
\end{cases}
```

Трансляция в Си:

```c
if (x < 0)  return pow(x, 2);
if (x == 0) return 0;
return sqrt(x);
```

### 8.2 `for..in`

```
for i in 1..n
    res = res + i^2
```
→ `for (int i = 1; i <= n; i++)`

С шагом:
```
for i in 0..100 step 2
```
→ `for (int i = 0; i <= 100; i += 2)`

### 8.3 `while`

```
while |x^2 - val| > 0.001
    x = (x + val / x) / 2
```

**Ограничения циклов:**
- `break` и `continue` отсутствуют
- Результат цикла не возвращается неявно — нужно явно указать переменную после блока цикла
- Переменные, изменённые внутри цикла, не выходят за пределы функции

### 8.4 Функциональные операторы суммы и произведения

**По диапазону индексов:**

| Синтаксис | Описание |
|-----------|---------|
| `\sum_{i=a}^{b} expr` | Сумма по индексу |
| `∑(i=a, b) expr` | Сумма по индексу (короткий синоним) |
| `\prod_{i=a}^{b} expr` | Произведение по индексу |
| `∏{i=a}{b} expr` | Произведение по индексу (короткий синоним) |

**По элементам массива (`\in`) — O(n):**

| Синтаксис | Описание |
|-----------|---------|
| `\sum_{x \in v} expr` | Сумма выражения по элементам `v` |
| `\prod_{x \in v} expr` | Произведение выражения по элементам `v` |
| `\min_{x \in v} expr` | Минимум выражения по элементам `v` |
| `\max_{x \in v} expr` | Максимум выражения по элементам `v` |

Парсер различает варианты по наличию `\in` vs `=` в нижнем индексе `_{}`.

**Функции-агрегаторы (один аргумент — массив):**

| Функция | Описание | Сложность |
|---------|---------|-----------|
| `sum(v)` | сумма элементов | O(n) |
| `product(v)` | произведение элементов | O(n) |
| `mean(v)` | среднее значение | O(n) |
| `std(v)` | стандартное отклонение | O(n) |
| `min(v)` | минимальный элемент | O(n) |
| `max(v)` | максимальный элемент | O(n) |

Dispatch `min`/`max` по числу аргументов: `min(v)` → агрегатор; `min(a, b)` → `fmin(a, b)`.

## 9. Блок `where`

Локальные определения и guard-условия под основной формулой.

### 9.1 Синтаксис

Строки в `where`-блоке бывают двух видов:

| Форма | Семантика |
|-------|-----------|
| `имя = выражение` | Определение локальной переменной |
| `выражение` (без `=`) | Guard clause — если ложно, функция возвращает `nan` |

```
force(m1, m2, r) = G * \frac{m1 * m2}{r^2}
    where
        m1 > 0
        m2 > 0
```

Трансляция в Си:

```c
mc_num force(mc_num m1, mc_num m2, mc_num r) {
    if (!(m1 > 0)) return NAN;
    if (!(m2 > 0)) return NAN;
    return G * (m1 * m2 / pow(r, 2));
}
```

### 9.2 Правила

- `where`-блок выполняется **до** основного выражения
- **Определения переменных** (`=`) — вычисляются в порядке топологической сортировки по зависимостям; пользователь не обязан следить за порядком объявления
- **Guard-и** (без `=`) — выполняются строго **сверху вниз** в том порядке, в котором написаны
- Guard-и выполняются после всех переменных, от которых зависят
- Циклические зависимости между переменными → **ошибка компиляции E033** (`CircularWhereDep`)
- Переменные из `where` видны только внутри функции
- Переопределение встроенной константы в `where` допускается, но вызывает предупреждение компилятора

**Пример порядка генерации:**

```
f(m, x) = E + offset
    where
        m > 0
        E = k * x^2
        k = m * 9.81
        E > 0
```

```c
mc_num f(mc_num m, mc_num x) {
    if (!(m > 0)) return NAN;       // guard 1 — первый по порядку
    mc_num k = m * 9.81;            // k нужна для E — идёт раньше
    mc_num E = k * pow(x, 2);       // E зависит от k
    if (!(E > 0)) return NAN;       // guard 2 — второй по порядку
    return E + offset;
}
```

## 12. Сложные математические операции

### 12.1 Пределы

Синтаксис:
```
val = \lim{x \to 0} \frac{sin(x)}{x}
L = \lim{x \to \infty} \frac{1}{x}
```

Реализация:
- **MVP:** только численная — вычисление в точке `a + 1e-9`; для `x → ∞` подставляется `1e15`
- **v2:** аналитическая (compile-time) для простых случаев (например, `lim x→∞ 1/x = 0.0`)

### 12.2 Производные

```
\frac{d}{dx} f(x)
```

Реализация:
- **MVP:** численная разностная схема `(f(x+h) - f(x)) / h`, `h = 1e-7`
- **v2:** аналитическая таблица правил (`d/dx sin(x)` → `cos(x)`, правило Лейбница и т.д.)

### 12.3 Интегралы

```
area = \int{0}{π} sin(x) dx
```

Реализация:
- **MVP:** численная, метод Симпсона, N=1000 шагов: `h = (b-a)/1000`
- **v2:** аналитическая таблица первообразных

Аналитическая (через таблицу первообразных):

| Выражение | Первообразная |
|-----------|--------------|
| `x^n` | `pow(x, n+1) / (n+1)` (n ≠ -1) |
| `1/x` | `log(abs(x))` |
| `e^x` | `exp(x)` |
| `\sin{x}` | `-cos(x)` |
| `\cos{x}` | `sin(x)` |
| `\frac{1}{x^2 + a^2}` | `(1/a) * atan(x/a)` |

## 13. Приоритеты операций

От высшего к низшему:

1. Унарные (`-`, `!`, `¬`, `not`)
2. Степень (`^`, `**`)
3. Умножение, деление, остаток, поэлементное умножение (`*`, `⋅`, `/`, `÷`, `%`, `.*`)
4. Векторные произведения (`⨯`)
5. Сложение, вычитание (`+`, `-`)
6. Сравнение (`<`, `>`, `<=`, `>=`, `==`, `!=`, `<>`, `≤`, `≥`)
7. Принадлежность (`∈`, `∉`, `in`, `!in`)
8. Логическое И (`and`, `&&`, `∧`)
9. Логическое ИЛИ (`or`, `||`, `∨`)
10. XOR (`xor`, `⊕`)

Примеры: `a + b ⋅ c` → `a + (b ⋅ c)` (скаляры: умножение); `a + v1 ⋅ v2` → `a + dot(v1, v2)` (векторы)

## 14. Экспорт

Все функции экспортируются по умолчанию (видны в сгенерированном `.h`-файле).

Функция считается **приватной** (не попадает в `.h`) если её имя начинается с `_`:

```
// Публичная — экспортируется
get_pos(pos, prev_pos, acc, dt) =
    pos + (pos - prev_pos) + acc * dt^2

// Приватная — только для внутреннего использования
_clamp(x, lo, hi) =
    if x < lo
        lo
    else if x > hi
        hi
    else
        x
```

## 15. Импорты и модули

### 15.1 Импорт пользовательских `.mc`-файлов

Компилятор разрешает импорты **статически**, на этапе компиляции. Все импортируемые файлы сливаются в одну программу (inline expansion). Результат — единый самодостаточный `.c` + `.h`, без внешних зависимостей. Это обязательно для embedded-платформ (STM32, Arduino), где нет динамической линковки.

#### Три формы импорта

**Полный импорт** — все публичные определения:
```
import "./constants.mc"
```

**Именованный импорт** — только указанные символы (статическая DCE):
```
from "./fluid_dynamics.mc" import drag_coefficient
from "./fluid_dynamics.mc" import drag_coefficient, lift_force
```

**Алиас** — все публичные определения с пространством имён:
```
import "./astronomy.mc" as astro
import "./ballistics.mc" as ball

res = astro.calculate(x) + ball.calculate(y)
```

`astro.calculate(x)` компилируется в вызов `astro__calculate(x)` (мангл через `__`).

#### Dependency graph

- Компилятор строит граф зависимостей и выполняет топологическую сортировку
- **Кэш**: каждый файл парсится и включается ровно один раз (diamond dependency — без дублирования)
- **Циклические импорты** запрещены → `ImportCycleError` с описанием цикла
- Файл не найден → `ImportNotFoundError`

#### Правила

- Путь всегда относительный: `"./"` или `"../"`
- Приватные функции (с `_`) не импортируются ни в какой форме
- `import std.*` — зарезервировано для стандартной библиотеки (v2)

### 15.2 Импорт стандартной библиотеки (v2)

```
import std.math
import std.physics
import std.physics.mechanics as mech
import std.geom.2d as geom
```

Стандартная библиотека в MVP не реализована. Встроенные математические функции (`sin`, `cos`, `sqrt`, …) и константы (`π`, `e`, `τ`) доступны без импорта всегда.

## 18. Поддержка размерностей (Units)

Размерности — синтаксический сахар для проверки совместимости на этапе компиляции. В Си превращаются в обычные числа (zero-cost).

```
gravity(m1: kg, m2: kg, r: m) =
    G * \frac{m1 * m2}{r^2}
```

**Базовые единицы СИ:** `kg`, `m`, `s`, `A`, `K`, `mol`

Производные единицы выводятся автоматически из операций:
- `m / s` → скорость
- `kg * m / s^2` → сила (эквивалент `N`)

Несовместимые операции → **ошибка компиляции:**
```
10kg + 5m   // ОШИБКА: несовместимые единицы
```

Научные префиксы (умножение на степень 10):
- `10k` → `10 * 1e3`
- `5m` → `5 * 1e-3` (милли)
- `2u` → `2 * 1e-6` (микро)
- `3n` → `3 * 1e-9` (нано)

## 20. Грамматика (PEG)

Грамматика описана в нотации PEG (Parsing Expression Grammar). Парсер строится как рекурсивный нисходящий (recursive descent) — PEG однозначно описывает приоритеты и не требует отдельных таблиц.

### 20.1 Лексические правила разрешения неоднозначностей

**Правило 1: `\lfloor`, `\rfloor`, `\lceil`, `\rceil`, `\lVert`, `\rVert`**

Лексер переводит LaTeX-команды в Unicode-токены до парсинга:

| LaTeX | Unicode-токен |
|-------|--------------|
| `\lfloor` | `⌊` |
| `\rfloor` | `⌋` |
| `\lceil` | `⌈` |
| `\rceil` | `⌉` |
| `\lVert` | `‖` (открывающий) |
| `\rVert` | `‖` (закрывающий) |

Парсер работает только с Unicode-токенами.

**Правило 2: `\sigma` и `\Gamma` — функция vs идентификатор**

Обе команды перегружены: могут быть греческой буквой-идентификатором или математической функцией.

| Контекст | Трактовка |
|---------|-----------|
| `\sigma{expr}` — за командой сразу `{` | функция `std(expr)` |
| `\sigma` без `{` | идентификатор `__uni_sigma` |
| `\Gamma{expr}` — за командой сразу `{` | функция `tgamma(expr)` |
| `\Gamma` без `{` | идентификатор `__uni_Gamma` |

Лексер применяет 1-символьный lookahead: следующий токен `{` → функция, иначе → идентификатор.

**Правило 3: `\text{name}` и `\operatorname{name}` — вызов функции по имени**

Обе команды извлекают имя из фигурных скобок и эмитируют токен `Identifier(name)`. Это позволяет писать LaTeX-style вызовы функций:

```
\text{sgn}(x)         // → sgn(x) → mc_sgn(x)
\operatorname{erf}(x) // → erf(x)
```

Если `\text{if}` или `\text{else}` — эмитируется соответствующий keyword-токен.

```peg
file        = (const_def / import / func_def / NL)*
import      = "import" identifier ("as" identifier)? NL
const_def   = identifier "=" literal NL

func_def    = identifier "(" params? ")" "=" (expr NL / NL INDENT block DEDENT)
params      = param ("," param)*
param       = identifier (":" type)? ("=" expr)?
type        = "num" ("[]")* / "int"

block       = stmt+
stmt        = where_block / for_stmt / while_stmt / if_stmt / expr NL
where_block = INDENT "where" NL INDENT where_line+ DEDENT DEDENT
where_line  = (identifier "=" expr / expr) NL

if_stmt     = "if" expr (expr NL / NL INDENT block DEDENT) ("else" (if_stmt / NL INDENT block DEDENT))?
for_stmt    = "for" identifier "in" expr ".." expr ("step" expr)? NL INDENT block DEDENT
while_stmt  = "while" expr NL INDENT block DEDENT

expr        = xor_expr
xor_expr    = or_expr ("xor" / "⊕" or_expr)*
or_expr     = and_expr (("or" / "||" / "∨") and_expr)*
and_expr    = in_expr (("and" / "&&" / "∧") in_expr)*
in_expr     = cmp_expr (("∈" / "∉" / "in" / "!in") range_or_set)?
              // range_or_set: "[" expr "," expr "]" (закрытый), "(" expr "," expr ")" (открытый),
              //               ℕ / ℤ / ℝ / ℚ / ℂ / identifier
cmp_expr    = add_expr (cmp_op add_expr)*          // цепочки: a < x < b
cmp_op      = "<" / ">" / "<=" / ">=" / "==" / "!=" / "<>" / "≤" / "≥"
add_expr    = vec_expr (("+" / "-") vec_expr)*
vec_expr    = mul_expr (("⨯") mul_expr)*
mul_expr    = pow_expr (("*" / "⋅" / "/" / "÷" / "%" / ".*") pow_expr)*
pow_expr    = unary ("^" / "**" pow_expr)?         // правоассоциативный
unary       = ("-" / "!" / "¬" / "not")? postfix
postfix     = primary ("!" / "°")? ("[" expr "]")* ("." identifier)*
              // "!" — факториал (постфикс); "°" — градусы → радианы
primary     = "(" expr ")"
            / abs_expr
            / norm_expr
            / floor_expr
            / ceil_expr
            / pm_expr
            / cases_expr
            / sum_expr
            / latex_expr
            / func_call
            / literal
            / identifier

abs_expr    = "|" expr "|"
norm_expr   = ("‖" / "\\lVert") expr ("‖" / "\\rVert")
floor_expr  = ("⌊" / "\\lfloor") expr ("⌋" / "\\rfloor")
ceil_expr   = ("⌈" / "\\lceil") expr ("⌉" / "\\rceil")
pm_expr     = ("\\pm" / "\\mp") expr           // \pm → [+x,-x]; \mp → [-x,+x]; возвращает num[2]
cases_expr  = "\\begin{cases}" NL
              (expr "&" expr "\\\\" NL)+       // случаи с условием
              (expr NL)?                       // опциональный else
              "\\end{cases}"
sum_expr    = ("\\sum" / "\\prod" / "\\min" / "\\max") "_{"
              ( identifier "=" expr "}" "^{" expr "}"   // по диапазону: i=a, верхняя граница b
              / identifier "\\in" identifier "}"        // по массиву: x \in v
              ) expr
latex_expr  = "\\" identifier ("{" expr "}")* ("[" expr "]" "{" expr "}")?
func_call   = identifier "(" (expr ("," expr)*)? ")"
literal     = number / array_lit / matrix_lit
array_lit   = "[" (expr ("," expr)*)? "]"
matrix_lit  = "[" NL? (array_lit NL?)+ "]"
number      = [0-9]+ ("." [0-9]+)? (("e" / "E") ("+" / "-")? [0-9]+)?
identifier  = [a-zA-Z_\u0080-\uFFFF] [a-zA-Z0-9_\u0080-\uFFFF]*
```

## 21. Компилятор: Pipeline

1. **Scanner** — токенизация, распознавание `Indent`/`Dedent`, Unicode-идентификаторов
2. **Lexer** — таблица синонимов (tg→tan, sh→sinh, …), трансляция `\команд` LaTeX
3. **Math-Parser** — вставка неявного умножения, нормализация `^` → `pow()`, цепочки сравнений
4. **ASI** — автоматическая расстановка `;`
5. **AST Builder** — построение дерева разбора
6. **CAS Engine** — symbolic lookup: интегралы, производные, алгебраические тождества, тригонометрические упрощения *(MVP: только `e^x` → `exp(x)`)*
7. **Constant Folder** — свёртка константных выражений
8. **Type Checker** — вывод типов, проверка размерностей units
9. **C-Generator** — генерация `.c` + `.h` файлов

**C-Generator: детали реализации**

- **Взаимная рекурсия:** перед телами всех функций генерируются forward-declaration прототипы — это позволяет функциям ссылаться друг на друга в любом порядке объявления
- **where-блок:** переменные топологически сортируются по зависимостям; циклическая зависимость → ошибка E033
- **`--target avr`:** добавляет `#define MC_AVR_TARGET 1` и fallback-определения `INFINITY`, `NAN`, `isnan`, `isinf`, `isfinite` через `__builtin_*`; по умолчанию `--precision f32`
- **Runtime helpers:** все вспомогательные функции (`mc_dot`, `mc_cross3`, `mc_matmul`, `mc_add_arr`, `mc_sub_arr`, `mc_mul_arr`, `mc_scale`, `mc_sgn`, `mc_gcd`, `mc_lcm`, `mc_binom`, `mc_factorial`, …) эмитируются как `static inline` в начале `.c`-файла; компилятор C DCE-удаляет неиспользуемые

### 21.1 CAS Engine (порядок применения)

1. Pattern Matcher — сравнение AST-узла с таблицей аналитических правил
2. Transformer — замена узла на готовый результат
3. Fallback — если правило не найдено, используется численный метод в рантайме

### 21.2 Алгебраические упрощения

| Правило | Пример |
|---------|--------|
| Разность квадратов | `a^2 - b^2` → `(a-b)(a+b)` |
| Логарифм произведения | `log(a) + log(b)` → `log(a*b)` |
| Тригонометрическое тождество | `sin(x)^2 + cos(x)^2` → `1.0` |
| Квадрат расстояния | `dist(A,B)^2` → убирает `sqrt` |

## 25. Справочник LaTeX-синтаксиса

Правило лексера: если токен начинается с `\`, слеш отбрасывается, идентификатор ищется в таблице ниже. Если совпадение найдено — применяется трансляция. Если нет — ошибка компиляции.

### 25.1 Арифметика

| LaTeX | Синоним без `\` | Трансляция в Си | Примечание |
|-------|----------------|----------------|-----------|
| `\frac{a}{b}` | — | `((a) / (b))` | Два блока `{}` обязательны |
| `x^{n}` | `x^n`, `x**n` | `pow(x, n)` | Правоассоциативный |
| `x^{a+b}` | — | `pow(x, (a+b))` | Блок `{}` для сложного выражения |
| `\sqrt{x}` | — | `sqrt(x)` | |
| `\sqrt[n]{x}` | — | `pow(x, 1.0/n)` | `n` — любое выражение |
| `\abs{x}` | `abs(x)`, `\|x\|` | `fabs(x)` | |
| `\lfloor x \rfloor` | `floor(x)` | `floor(x)` | Скобки как единый токен |
| `\lceil x \rceil` | `ceil(x)` | `ceil(x)` | Скобки как единый токен |
| `\lVert v \rVert` | `norm(v)` | `mc_norm(v, v_len)` | Норма вектора |
| `n!` | `factorial(n)` | цикл или lookup | Постфиксный оператор |
| `\binom{n}{k}` | `binom(n, k)` | `factorial(n)/(factorial(k)*factorial(n-k))` | |
| `\pm expr` | `±expr` | `[+expr, -expr]` | Возвращает `num[2]` |
| `\mp expr` | `∓expr` | `[-expr, +expr]` | Возвращает `num[2]`, обратный порядок |
| `a \mod b` | `a % b`, `mod(a, b)` | `fmod(a, b)` | |
| `30°` | — | `30 * (π / 180)` | Символ `°` (U+00B0) |

По диапазону индексов:

| LaTeX | Синоним | Трансляция в Си |
|-------|---------|----------------|
| `\sum_{i=a}^{b} expr` | `∑(i=a, b) expr` | `for`-цикл с аккумулятором |
| `\prod_{i=a}^{b} expr` | `∏(i=a, b) expr` | `for`-цикл с аккумулятором |

По элементам массива (`\in`):

| LaTeX | Синоним | Трансляция в Си |
|-------|---------|----------------|
| `\sum_{x \in v} expr` | `sum(v)` (если expr=x) | O(n) цикл по элементам |
| `\prod_{x \in v} expr` | `product(v)` (если expr=x) | O(n) цикл по элементам |
| `\min_{x \in v} expr` | `min(v)` (если expr=x) | O(n) цикл, возвращает минимум |
| `\max_{x \in v} expr` | `max(v)` (если expr=x) | O(n) цикл, возвращает максимум |

Функции-агрегаторы: `sum(v)`, `product(v)`, `mean(v)` / `\bar{v}`, `std(v)` / `\sigma{v}`, `min(v)` / `max(v)`.

### 25.5 Математический анализ

| LaTeX | Трансляция | Метод |
|-------|-----------|-------|
| `\lim{x \to a} expr` | вычисление в `a + 1e-9` | численный |
| `\lim{x \to \infty} expr` | вычисление при `x = 1e15` | численный |
| `\frac{d}{dx} f(x)` | `(f(x+1e-7) - f(x)) / 1e-7` | конечные разности |
| `\int{a}{b} f(x) dx` | симметричная форма mclang | метод Симпсона |
| `\int_{a}^{b} f(x) dx` | стандартная LaTeX-форма | метод Симпсона |

### 25.6 Константы и греческие буквы

Встроенные константы (`π`, `τ`, `φ`, `e`) подставляются значением. Остальные греческие буквы — идентификаторы.

| LaTeX | Unicode | ASCII в Си |
|-------|---------|------------|
| `\pi` | `π` | `M_PI` (встроенная) |
| `\tau` | `τ` | `__uni_tau` (= 2π, встроенная) |
| `\phi` | `φ` | `__uni_phi` (= φ, встроенная) |
| `\infty` | `∞`, `inf` | `INFINITY` |
| `\alpha` … `\omega` | `α` … `ω` | `__uni_alpha` … `__uni_omega` |
| `\Gamma` … `\Omega` | `Γ` … `Ω` | `__uni_Gamma` … `__uni_Omega` |

Вариантные формы (`\varepsilon`, `\varphi`, `\vartheta`, `\varpi`, `\varrho`) — синонимы основных.

**Греческая буква + ASCII без пробела = один идентификатор:** `πd` → `__uni_pid`. Для умножения нужен пробел или `*`: `π d` или `π*d`.

### 25.7 Матричные операции

| LaTeX / Unicode | Синоним | Трансляция |
|----------------|---------|-----------|
| `A^{\top}`, `A^{T}` | `transpose(A)` | перестановка индексов |
| `A^{-1}` | `inv(A)` | обратная матрица (до 3×3) |
| `\det(A)` | `det(A)` | определитель (до 3×3) |
| `\cdot` | `⋅` | dispatch: `*` / `mc_dot` / `mc_matmul` |
| `\times` | `⨯` | векторное произведение `cross(a, b)` |
| `.*` | — | поэлементное произведение → `mc_mul_arr` |

### 25.8 Функции и операторы

| LaTeX | Синоним | Трансляция в Си |
|-------|---------|----------------|
| `\min(a,b)` / `\max(a,b)` | `min(a,b)` / `max(a,b)` | `fmin` / `fmax` |
| `\text{sgn}(x)` | `sgn(x)` | `mc_sgn(x)` |
| `\gcd(a,b)` | `gcd(a,b)` | `mc_gcd` |
| `\lcm(a,b)` | `lcm(a,b)` | `mc_lcm` |
| `\erf(x)` | `erf(x)` | `erf(x)` |
| `\Gamma{x}` | `gamma(x)` | `tgamma(x)` |
| `\bar{v}` | `mean(v)` | `mc_mean` |
| `\sigma{v}` | `std(v)` | `mc_std` |
| `\neq`, `\ne` | `!=`, `<>` | `!=` |
| `\leq` | `<=`, `≤` | `<=` |
| `\geq` | `>=`, `≥` | `>=` |
| `\wedge`, `\land` | `∧` | `&&` |
| `\vee`, `\lor` | `∨` | `\|\|` |
| `\neg`, `\lnot` | `¬` | `!` |

**Disambiguation `\sigma` и `\Gamma`:** лексер применяет lookahead — если за командой идёт `{`, это функция (`std(v)`, `tgamma(x)`); иначе — идентификатор (`__uni_sigma`, `__uni_Gamma`).

### 25.9 Автоматически масштабируемые скобки

`\left`/`\right` — размер игнорируется, остаётся тип скобки: `\left( \right)` → `(…)`, `\left[ \right]` → `[…]`, `\left\|…\right\|` → `fabs(…)`.

### 25.10 Не поддерживается (v2+)

`\int f dx` (неопределённый), `\partial`, `\nabla`, `\oint` — требуют CAS.

## 26. MVP: реализованные возможности

### Компилятор

- Лексер с поддержкой Unicode-идентификаторов и греческих букв
- Парсер (PEG, recursive descent)
- Построение AST
- Генератор Си-кода (`.c` + `.h`)
- Таргеты: `--target c`, `--target wasm`, `--target shared`, `--target rust`, `--target node`
- Флаг точности: `--precision f64` (default), `--precision f32`, `--precision fixed`

### Типы

- `num`, `int`
- `num[]` (вектор) + неявный `_len` параметр
- `num[][]` (матрица) + неявные `_rows`, `_cols` параметры
- Статические массивы `num[N]` → стек

### Синтаксис

- Объявление функций: `f(x) = expr`
- Глобальные константы: `G = 6.67e-11`
- Значимые отступы
- ASI (автоматические точки с запятой)
- Перенос строк через оператор или открытую скобку
- Параметры по умолчанию: `f(x, y = 0)`
- Неявный `return` (последнее выражение)
- `_` префикс → приватная функция
- Все функции экспортируются по умолчанию
- Комментарии `//` и `#`
- Импорт пользовательских `.mc`-файлов: `import "./file.mc"`, `from "./file.mc" import name`, `import "./file.mc" as alias`

### Математика

- Неявное умножение: `2x`, `2(a+b)`
- LaTeX: `\frac{a}{b}`, `\sqrt{x}`, `\sqrt[n]{x}`, `x^{n}`
- Степень: `x^n`, `x**n` → `pow(x, n)`
- Все тригонометрические функции + русские синонимы (`tg`, `ctg`, `sh`, `ch`, …)
- Логарифмы: `ln`, `log`, `log10`, `\lg`, `log{base}{x}`
- `exp(x)`, `e^x`, `abs(x)`, `|x|`, `\abs{x}`, `floor`, `ceil`, `round`
- Встроенные константы: `π`, `e`, `τ`, `φ`, `inf`, `nan`
- Физические константы: `G`, `c`, `h`, `k_B`, `N_A`, `R`

### Операторы

- Арифметика: `+`, `-`, `*`, `⋅`, `/`, `÷`, `%`, `.*`, `^`, `**`
- Сравнение: `<`, `>`, `<=`, `>=`, `==`, `!=`, `<>`, `≤`, `≥`
- Цепочки сравнений: `0 < x < 10`
- Логика: `and`, `or`, `xor`, `not`, `&&`, `||`, `!`, `∧`, `∨`, `⊕`, `¬`
- Принадлежность: `∈`, `∉`, `in`, `!in` + диапазоны `[a,b]`, `(a,b)` + множества `ℕ`, `ℤ`, `ℝ`
- Остаток: `%`, `mod(x, y)`
- Dispatch `⋅` по типу: скаляры → `*`, векторы → dot product, матрицы → матричное умножение
- Модуль `|x|` с контекстным разбором (`||` vs `|...|`)

### Управляющие конструкции

- `if / else` (инлайн + блочный), только `if-first`
- Guard clause: `if (cond) value` без `else`
- `for i in a..b`, `for i in a..b step n`
- `while expr`
- `∑`, `∏` (разворачиваются в `for`-цикл)

### Блок `where`

- Определения переменных (топологическая сортировка)
- Guard-и (порядок объявления, нарушение → `nan`)

### Векторы и матрицы

- Литералы: `[1, 2, 3]`, `[[1, 0], [0, 1]]`
- Поэлементные операции: `+`, `-`, `*` (на скаляр), `.*`
- Матричное умножение: `A ⋅ B`
- `dot(v1, v2)`, `cross(v1, v2)`
- Индексация: `v[i]`, `m[i][j]`, срезы `v[a..b]`, `m[:, j]`
- Функции: `transpose`, `det`, `inv`, `norm`
- Константы: `I(n)`, `zeros(r,c)`, `ones(r,c)`
- `.rows`, `.cols`

### Специальные состояния

- `inf`, `-inf`, `nan`
- `is_nan(x)`, `is_inf(x)`, `is_finite(x)`
- Деление на ноль → `inf`/`nan`, не падение

### Сложные операции (численные методы)

- Предел `\lim{x \to a}` → вычисление в точке `a ± 1e-9`
- Производная `\frac{d}{dx}` → разностная схема `(f(x+h) - f(x)) / h`
- Интеграл `\int{a}{b}` → метод Симпсона через указатель на функцию
- `solve(t, lo, hi) { expr == val }` → метод бисекции, точность `1e-9`
- Рекурсия (взаимная и хвостовая)

### CLI

- `mclang eval <file.mc>` — список экспортируемых функций
- `mclang eval <file.mc> "fn(arg1, arg2)"` — вызов функции с аргументами (требует GCC в PATH)

- Вывод производных единиц

### Модули

- `std.*` импорты с реальными данными

### Визуальный конструктор

Веб-приложение поверх рабочего компилятора. Требует стабильного Wasm-таргета.

**Редактор:**
- Рендеринг формул в реальном времени (LaTeX → MathML/Canvas)
- Палитра символов: греческие буквы, `∫`, `∑`, `∏`, `√`, `±`, …
- Автодополнение функций и констант

**Библиотеки:**
- Браузер модулей: `std.physics`, `std.chemistry`, `std.geom.*`
- Подключение одним кликом, автоимпорт

**Запуск:**
- Тестирование функций: задать аргументы → получить результат прямо в браузере (через Wasm)
- Симуляции: пошаговый запуск с визуализацией промежуточных значений
- Вывод результатов в таблицу

**Графики:**
- 2D-графики функций (y = f(x))
- Параметрические кривые
- Фазовые портреты для симуляций

**Визуальные блоки:**
- `if / else` — блок-схема
- `for`, `while` — визуальный цикл с диапазоном
- `where` — панель зависимостей и guard-ов
- `from "./file.mc" import` — дерево импортов

### Прочее

- `solve` без явного диапазона (дефолтный `[0, 1e9]`)
- Именованные таблицы с строковыми ключами (`"Al": 2700`)
- Comprehensions (`[[1 if i==j else 0 for i in 0..2] for j in 0..2]`)
- Научные префиксы (`10k`, `5m`, `2u`)
