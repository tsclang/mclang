# mclang — Контекст проекта для Claude

## Что это

**mclang** (Math C Language) — компилятор специализированного языка математических вычислений. Пользователь пишет формулы в декларативном стиле с поддержкой LaTeX-синтаксиса, Unicode-переменных и математических операторов. Компилятор транслирует это в чистый Си-код без зависимостей.

Целевая аудитория: физики, инженеры, геймдев-разработчики.
Целевые платформы: PC (JS/Wasm, Python, C/C++), микроконтроллеры (STM32, Arduino).

## Ключевые решения (зафиксированы в SPEC.md)

### Синтаксис
- Функции: `f(x) = выражение` — декларативный стиль
- Значимые отступы (Python-style), ASI (авто `;`)
- Только `if-first` форма: `if (cond) val else val2`
- Неявный `return` — последнее выражение
- `_имя` → приватная функция (не экспортируется)

### Типы
- `num` — универсальное число (→ `double` / `float` / `fixed` в зависимости от таргета)
- `int` — только для индексов и счётчиков
- `num[]` → `mc_num* v, int v_len` (неявный параметр длины)
- `num[][]` → `mc_num* m, int m_rows, int m_cols`
- `.length` для матриц запрещён — только `.rows` и `.cols`

### Блок `where`
- Строки с `=` — определения переменных (топологическая сортировка)
- Строки без `=` — guard clause → `if (!cond) return NAN;`
- Guard-и выполняются в порядке объявления

### Мутабельность
- Параметры функции — иммутабельны (ошибка компиляции при переназначении)
- Локальные переменные — мутабельны
- Локальная переменная должна быть инициализирована перед первым использованием

### Экспорт
- Все функции экспортируются по умолчанию
- `_имя` → не попадает в `.h`-файл
- Ключевого слова `export` нет

### `|...|` vs `||`
- Disambiguация по контексту предыдущего токена
- VALUE-контекст (после числа/идентификатора/`)`): `|` = закрывающий модуль или `||`
- OPERATOR-контекст (после оператора/`(`/ключевого слова): `|` = открывающий модуль

### `⋅` — dispatch по типу
- `num ⋅ num` → `*`
- `num[] ⋅ num[]` → dot product
- `num[][] ⋅ num[][]` → матричное умножение

### Множественный возврат
- `\pm expr` → `num[2]` = `[+expr, -expr]`
- Явный: `f() = [a, b, c]`
- В Си: `static mc_num _result[N]; return _result;`

### `\sigma{v}` и `\Gamma{x}`
- Если за командой следует `{` → функция (`std(v)`, `tgamma(x)`)
- Если стоит одиноко → греческая буква-идентификатор

## Pipeline компилятора

```
Scanner → Lexer → Math-Parser → ASI → AST → CAS(v2) → ConstFold → TypeCheck → C-Generator
```

1. **Scanner** — токенизация, INDENT/DEDENT, Unicode
2. **Lexer** — таблица синонимов, LaTeX → Unicode-токены, disambiguation `|`
3. **Math-Parser** — неявное умножение, `^` → `pow()`, цепочки сравнений
4. **ASI** — авто `;`
5. **AST Builder**
6. **CAS Engine** — (MVP: пропускается, только численные методы)
7. **Constant Folder** — свёртка константных выражений
8. **Type Checker** — вывод типов, dispatch `⋅`, проверка размерностей
9. **C-Generator** — `.c` + `.h`

## Язык реализации компилятора

**TypeScript** (Node.js).

Структура проекта:
```
src/
  lexer/       — фаза 1
  parser/      — фаза 2
  ast/         — узлы AST
  codegen/     — фаза 3+
  math/        — фаза 4 (LaTeX, константы, constant folding)
  types/       — type checker
  cli/         — точка входа (аргументы командной строки)
tests/
  lexer/
  parser/
  codegen/
```

Точка входа CLI: `mclang <file.mc> --target <c|wasm|shared> [--precision <f64|f32|fixed>]`
